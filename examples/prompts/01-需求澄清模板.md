# 需求澄清模板

> 接到新需求时使用，确保理解正确

---

## 使用方法

```
把下面的模板复制给 AI，填充 [ ] 里的内容
```

---

## 模板内容

```markdown
## 背景
[为什么需要这个功能？用户遇到什么问题？]

## 目标
[通过这个需求，要解决什么问题/达成什么目的？]

## 功能需求
### 必须实现
- [ ] [需求点 1]
- [ ] [需求点 2]
- [ ] [需求点 3]

### 可选实现
- [ ] [锦上添花的功能点]

## 用户场景
- 场景 1：[描述用户在这个场景下怎么用]
- 场景 2：[描述另一个典型场景]

## 验收标准
- [ ] [标准 1：功能表现]
- [ ] [标准 2：性能要求]
- [ ] [标准 3：兼容性要求]

## 约束条件
- 技术限制：[比如必须兼容 iOS 15+]
- 时间限制：[比如必须在 XX 日期前完成]
- 其他限制：[比如不能调用付费 API]

## 参考资料
- [相关文档或链接]
- [类似功能的参考]
```

---

## 示例

```markdown
## 背景
用户反馈在商品列表页找不到想买的东西，搜索功能不好用。

## 目标
改进搜索功能，让用户能更快找到商品。

## 功能需求
### 必须实现
- [ ] 支持按商品名称搜索
- [ ] 支持按价格区间筛选
- [ ] 显示搜索历史

### 可选实现
- [ ] 语音搜索
- [ ] 拍照搜索

## 用户场景
- 场景 1：用户知道想买什么，直接搜名称
- 场景 2：用户想找特定价位的商品

## 验收标准
- [ ] 搜索结果 1 秒内显示
- [ ] 支持中英文搜索
- [ ] 无结果时显示友好提示

## 约束条件
- 必须兼容 iOS 14+
- 不能调用付费 API
- 必须在 3 月 1 日前完成

## 参考资料
- 竞品 A 的搜索功能截图
- 搜索算法文档：xxx
```

---

## 提示

1. **不要怕写得多**：信息越多，AI 理解越准确
2. **分清必须和可选**：避免 scope creep
3. **写清楚验收标准**：避免交付时扯皮
4. **给参考资料**：帮助 AI 理解上下文
