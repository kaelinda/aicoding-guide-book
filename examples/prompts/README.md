# Prompt 模板库

> 面向 AI 编程常见任务的可复用 Prompt 模板

## 🎯 为什么要做 Prompt 模板库

很多人在使用 AI 编程工具时，效果不稳定，并不是因为模型不够强，而是因为：

- 每次都临时想怎么问
- 同一个任务反复换说法
- 没有形成固定的表达结构
- 遇到复杂任务时不知道该先讲什么

Prompt 模板库的价值，不是让你“机械套话”，而是帮你建立一套更稳定的表达方式。

## ✅ 这套模板适合用来做什么

这套模板主要面向以下场景：

- 需求澄清
- 从零生成功能
- Bug 排查
- 重构优化
- 测试生成
- 文档补充
- 代码审查

你不需要一次掌握所有模板，可以先从最常用的两类开始：

1. 需求澄清
2. Bug 排查

## 🧭 使用模板时的 4 个原则

### 1. 先交代上下文
AI 在不了解上下文时，很容易做出“看起来通用，但并不贴项目”的回答。

所以建议先交代：

- 技术栈
- 当前文件或模块
- 你现在在做什么
- 你的目标是什么

### 2. 再说明目标
不要只说“帮我处理一下”。

更好的方式是明确写出：

- 我希望得到什么
- 结果应该长什么样
- 优先级是什么

### 3. 再给约束
这是非常重要的一步。

例如：

- 不要重写整个项目
- 保持初学者易读
- 优先最小修改
- 先解释再写代码
- 不要引入额外依赖

### 4. 最后给验收标准
验收标准越清楚，AI 越容易给出更接近你预期的结果。

例如：

- 页面能运行
- 搜索结果正确
- 删除不会删错对象
- 代码结构比现在更清晰

## 💡 推荐使用方式

你不需要逐字照抄模板。

更推荐的方式是：

1. 先复制模板
2. 把其中的变量替换成你当前项目的信息
3. 删除不必要的部分
4. 保留最关键的上下文、目标、约束、验收标准

模板是“骨架”，不是“标准答案”。

## ⚠️ 使用模板时最容易犯的错误

### 错误 1：模板太长，但没有重点
Prompt 不是越长越好。

如果写了很多信息，但没有突出目标，AI 还是可能抓不到重点。

### 错误 2：没有交代限制条件
比如你明明只想做最小修改，却没说，AI 很可能直接大改结构。

### 错误 3：没有给验收标准
没有验收标准时，AI 会默认自己理解“什么算完成”。

这往往就是误差来源。
