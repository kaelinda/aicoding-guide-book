# AGENTS.md - AI Coding Guide Book

> 本文件指导 AI 助手如何理解和维护这个项目

## 📚 项目概述

这是一个面向**初级 iOS 开发者**的 AI 编程入门教程项目。目标读者：
- 有基础的 iOS 开发经验（了解 Swift、Xcode）
- 对 AI 编程感兴趣，但不知从何入手
- 希望提升开发效率的独立开发者

## 🎯 项目目标

1. **系统化** - 从基础到高级，循序渐进
2. **实用性** - 每个知识点都有实际案例
3. **时效性** - 保持内容更新，紧跟技术发展
4. **可操作** - 提供可执行的步骤和命令

## ✍️ 写作规范

### 文档风格

- **简洁有力** - 避免废话，直击要点
- **结构清晰** - 使用标题、列表、表格组织内容
- **示例丰富** - 每个概念都配实际案例
- **图文并茂** - 适当使用图表、代码块、截图

### 代码规范

- 使用代码块标记语言：` ```swift `, ` ```bash `
- 添加注释解释关键代码
- 提供完整可运行的示例
- 标注代码来源（如果是引用）

### 文件命名

- 使用中文命名文件夹：`基础篇/`、`进阶篇/`
- 使用数字排序：`01-环境搭建/`、`02-工具安装/`
- README.md 作为章节入口

### 章节结构

每个章节包含：

```markdown
# 章节标题

> 一句话概述

## 🤔 是什么（概念）

## 🎯 为什么重要

## 🛠 怎么做（实践步骤）

## 💡 最佳实践

## ⚠️ 常见问题

## ✅ 小结

## 🔗 相关资源
```

## 🔧 维护指南

### 更新内容

1. **新增章节**
   - 确定所属篇章（基础/进阶/实战/大师）
   - 按章节结构编写
   - 更新上级 README 的目录链接

2. **更新现有内容**
   - 检查技术是否过时
   - 更新命令、链接
   - 添加新的最佳实践

3. **添加示例代码**
   - 放在 `docs/examples/` 目录
   - 在文档中引用链接（GitBook 内容根是 `docs/`，模板必须在其下才会被收录）

### 质量检查

- [ ] 所有链接可访问
- [ ] 代码示例可运行
- [ ] 无错别字
- [ ] 符合写作规范
- [ ] 目录链接正确
