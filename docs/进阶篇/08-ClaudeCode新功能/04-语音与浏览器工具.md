## VOICE_MODE 语音控制

### 什么是 VOICE_MODE

VOICE_MODE 是 Claude Code 的语音控制功能，允许开发者通过语音与 AI 交互，彻底解放双手。当你双手在键盘上忙碌、不方便打字，或者想要更自然地与 AI 沟通时，VOICE_MODE 让这一切成为可能。

```bash
# 启动语音模式
/voice on

# 说话后，Claude Code 会：
# 1. 实时语音转文字
# 2. 分析指令意图
# 3. 执行相应操作
# 4. 语音播报结果（可选）
```

### 激活方式

| 方式 | 操作 |
|------|------|
| 快捷键 | `Cmd/Ctrl + Shift + V` |
| 命令 | `/voice on` |
| 鼠标 | 点击工具栏 🎤 图标 |

### 基础配置

```json
// .claude/voice.json
{
  "enabled": true,
  "language": "zh-CN",       // 语音识别语言
  "tts": {
    "enabled": true,         // 是否播报回复
    "voice": "female",       // female / male
    "speed": 1.0             // 语速
  },
  "wakeWord": "嘿 Claude",   // 唤醒词（可选）
  "continuous": false        // 持续监听模式
}
```

### 常用指令

```

"解释一下这段代码在做什么"

"下一步怎么走"
    ↓

"停止"
    ↓
```

## WEB_BROWSER_TOOL 浏览器工具

### 什么是 WEB_BROWSER_TOOL

WEB_BROWSER_TOOL 让 Claude Code 能够直接操作浏览器，访问网页、填写表单、点击按钮、抓取数据。这在以下场景特别有用：

- **数据采集**：抓取公开网页数据
- **自动化测试**：模拟用户操作进行测试
- **信息验证**：验证 API 返回的页面是否正常
- **文档获取**：获取在线文档和教程

### 基础使用

```bash
# 访问网页
/web open https://example.com

# 截取页面截图
/web screenshot

# 获取页面内容
/web content

# 在页面中搜索
/web find "登录"

# 点击元素
/web click "#login-button"

# 填写表单
/web fill "#username" "myuser"
/web fill "#password" "mypass"

/web submit "#login-form"
```

### 进阶操作

```bash
# 等待元素出现
/web wait "#dashboard" --timeout 10s

# 滚动页面
/web scroll down
/web scroll up 300px

# 切换标签页
/web switch-tab 2

# 执行 JavaScript
/web eval "document.title"
```

### 与 Agent 联动

```bash
# Agent 自动使用浏览器
"帮我查看 GitHub 上这个项目的 Star 数"

# Agent 会：
# 1. 使用 /web open 访问页面
# 2. 使用 /web content 提取数据
# 3. 汇总结果回复你
```

## 隐私与安全

### 语音数据

- 语音识别在本地处理，不上传到云端
- TTS 播报可根据需要关闭
- 唤醒词功能可选启用

### 浏览器数据

```json
{
  "browser": {
    "history": false,       // 不记录浏览历史
    "cookies": false,       // 不保存 Cookie
    "autoLogout": true,     // 离开时自动登出
    "sensitiveDomains": [   // 敏感域名
      "bank.com",
      "*.gov"
    ]
  }
}
```

## 常见问题

### Q: 语音识别准确吗？

A: VOICE_MODE 支持中文和英文，识别准确率在安静环境下可达 95% 以上。在嘈杂环境中建议使用耳机麦克风。

### Q: 浏览器工具能登录账号吗？

A: 支持登录，但建议在 `sensitiveDomains` 中配置敏感网站，防止误操作。

### Q: 语音模式会影响开发效率吗？

A: 语音适合非紧急、不需要精确输入的场景。大量代码编写仍建议使用键盘。
