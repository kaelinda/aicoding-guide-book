# 03-Prompt 技巧

> 好的 Prompt = 清晰的需求 + 明确的上下文 + 可执行的指令

## 🤔 为什么 Prompt 很重要

### 同一个需求，不同结果

```
❌ 糟糕的 Prompt：
"写个页面"

AI：[困惑] 什么页面？什么功能？用什么技术？

结果：生成的代码不符合预期，需要反复修改
```

```
✅ 好的 Prompt：
"创建一个用户登录页面，包含：
- 邮箱输入框（带格式验证）
- 密码输入框（至少 8 位，包含数字和字母）
- 登录按钮（点击后调用 API）
- 使用 SwiftUI 实现
- 遵循 MVVM 架构"

AI：[理解需求] [生成符合要求的代码]

结果：一次到位，无需修改
```

**核心原则：AI 的输出质量，取决于你的输入质量。**

## 🎯 Prompt 基础结构

### 五要素法

```
1. 角色（Role）
   "你是一个 iOS 开发专家..."

2. 任务（Task）
   "创建一个用户列表页面..."

3. 上下文（Context）
   "项目使用 SwiftUI + MVVM 架构..."

4. 约束（Constraints）
   "代码需要通过 SwiftLint 检查..."

5. 示例（Example）
   "参考这个格式：{示例代码}"
```

### 实战示例

```
【角色】你是一个有 10 年经验的 iOS 开发专家，精通 SwiftUI 和 MVVM 架构。

【任务】创建一个商品列表页面。

【上下文】
- 项目名称：ShoppingApp
- 技术栈：Swift 5.9, SwiftUI, Combine
- 架构：MVVM
- 网络层：使用 URLSession + async/await

【约束】
- 遵循 Swift API 设计指南
- 所有 public 方法添加文档注释
- 使用 @MainActor 标记 UI 更新
- 错误处理使用 Result 类型

【示例】
参考格式：
struct ProductListView: View {
    @StateObject private var viewModel: ProductListViewModel
    
    var body: some View {
        // 实现
    }
}
```

## 💡 核心技巧

### 技巧 1：具体化

```
❌ 抽象：
"优化一下性能"

✅ 具体：
"优化商品列表页面的滚动性能：
- 当前问题：滚动时有明显卡顿
- 数据量：约 100 个商品
- 每个商品包含：图片（200x200）、标题、价格
- 目标：60fps 流畅滚动"
```

### 技巧 2：场景化

```
❌ 孤立：
"写一个验证函数"

✅ 场景化：
"用户注册场景下的邮箱验证：
- 用户在注册页面输入邮箱
- 点击注册按钮时验证
- 验证失败显示红色错误提示
- 验证通过才能提交表单"
```

### 技巧 3：示例驱动

```
❌ 描述：
"写一个像微信那样的聊天界面"

✅ 示例驱动：
"参考微信聊天界面的布局：
- 顶部：对方头像和昵称
- 中间：消息列表（气泡样式）
  - 我的消息：绿色气泡，右对齐
  - 对方消息：白色气泡，左对齐
- 底部：输入框 + 发送按钮

请实现类似的 SwiftUI 视图"
```

### 技巧 4：分步骤

```
❌ 一次性：
"开发一个完整的电商 App"

✅ 分步骤：
"第一步：创建项目基础架构
第二步：实现用户模块
第三步：实现商品模块
第四步：实现订单模块"

每个步骤单独给出 Prompt，迭代完成。
```

### 技巧 5：提供反馈

```
第一轮：
你："创建一个用户列表页面"
AI：[生成代码]

第二轮：
你："样式不太对，改用卡片式布局，
    每个卡片包含头像、用户名、简介"

AI：[调整代码]

第三轮：
你："这个可以了，继续下一个功能"
```

## 🛠 实战模板

### 模板 1：创建新功能

```markdown
## 功能描述
[功能名称]

## 用户故事
作为 [角色]，我希望 [行为]，以便 [收益]

## 技术要求
- 技术栈：[Swift/SwiftUI/等]
- 架构：[MVVM/Clean/等]
- 数据源：[API/本地数据库/等]

## UI 要求
- 布局：[描述或截图]
- 交互：[点击、滑动等]
- 动画：[是否需要]

## 验收标准
- [ ] 功能 1
- [ ] 功能 2
- [ ] 测试通过

## 参考
[相似功能的链接或截图]
```

### 模板 2：修复 Bug

```markdown
## Bug 描述
[具体现象]

## 复现步骤
1. 打开 [页面]
2. 点击 [按钮]
3. 观察 [现象]

## 期望行为
[应该是什么样的]

## 实际行为
[实际是什么样的]

## 错误信息
```
[错误日志]
```

## 相关代码
文件：[文件路径]
方法：[方法名]

## 环境
- iOS 版本：
- 设备型号：
- App 版本：
```

### 模板 3：代码重构

```markdown
## 重构目标
[为什么要重构]

## 当前代码
[当前实现方式]

## 问题
- [问题 1]
- [问题 2]

## 目标架构
[期望的架构方式]

## 约束
- 保持功能不变
- 通过现有测试
- 不破坏 API 兼容性

## 验收标准
- [ ] 代码结构清晰
- [ ] 测试全部通过
- [ ] 性能未降低
```

### 模板 4：性能优化

```markdown
## 性能问题
[具体表现]

## 性能指标
- 当前：[fps/响应时间/内存占用]
- 目标：[期望值]

## 分析
- 瓶颈位置：[文件名:行号]
- 可能原因：[分析]

## 优化方案
[建议的优化方式]

## 验证方法
[如何验证优化效果]
```

## 🎯 针对不同场景的 Prompt 策略

### 场景 1：快速原型

```
策略：简化 Prompt，快速迭代

你："快速实现一个登录页面的原型"
AI：[生成基础代码]
你："添加忘记密码功能"
AI：[补充功能]
你："优化一下样式"
AI：[调整样式]
```

### 场景 2：生产代码

```
策略：详细 Prompt，严格验收

你："创建生产级别的登录页面：
    1. 邮箱验证（正则表达式）
    2. 密码强度检查（至少 8 位，包含大小写和数字）
    3. 错误处理（网络异常、服务器错误）
    4. 加载状态显示
    5. 单元测试覆盖
    6. 符合 SwiftLint 规范
    7. 添加文档注释"

AI：[生成完整、高质量的代码]
```

### 场景 3：学习探索

```
策略：询问 + 验证

你："SwiftUI 中有哪些动画方式？
    请举例说明每种方式的适用场景"

AI：[列出所有动画方式 + 示例]

你："哪个方式适合做页面切换动画？"

AI：[推荐最适合的方式]
```

## ⚠️ 常见错误

### 错误 1：太简略

```
❌ "写个页面"
✅ "创建一个商品详情页面，包含图片轮播、标题、价格、购买按钮"
```

### 错误 2：太模糊

```
❌ "优化一下"
✅ "优化列表滚动性能，目标 60fps，当前只有 30fps"
```

### 错误 3：缺少上下文

```
❌ "修复这个 Bug"
✅ "修复登录页面的邮箱验证 Bug：
    - 问题：邮箱包含 . 时验证失败
    - 文件：LoginViewModel.swift
    - 方法：validateEmail"
```

### 错误 4：期望过高

```
❌ "开发一个像微信的 App"
✅ "开发一个聊天 App，第一步先实现消息列表页面"
```

### 错误 5：不提供反馈

```
❌ 生成代码后直接用
✅ 审查代码，提出改进建议，迭代优化
```

## 📊 Prompt 质量评估

### 自检清单

```
□ 是否明确了要做什么？
□ 是否说明了为什么做？
□ 是否提供了足够的上下文？
□ 是否设定了约束条件？
□ 是否定义了验收标准？
□ 是否提供了示例或参考？
□ 是否考虑了边界情况？
□ 是否可验证结果？
```

### 质量等级

```
Level 1 - 可用：
- 基本完成任务
- 需要多次修改

Level 2 - 良好：
- 一次完成 80%
- 需要少量调整

Level 3 - 优秀：
- 一次完成 95%
- 符合所有要求

Level 4 - 完美：
- 一次完成 100%
- 超出预期
```

## 🎓 进阶技巧

### 技巧 1：链式 Prompt

```
第一轮：
你："分析这个项目的架构"

AI：[分析结果]

第二轮：
你："基于这个架构，设计用户模块"

AI：[设计方案]

第三轮：
你："按照这个方案，实现登录功能"

AI：[实现代码]
```

### 技巧 2：角色扮演

```
你："你现在是我团队的 Tech Lead，
    请审查这段代码并给出改进建议"

AI：[从 Tech Lead 视角审查]

你："现在你是产品经理，
    这个功能是否符合用户需求？"

AI：[从产品视角评估]
```

### 技巧 3：反向提问

```
你："我想实现一个用户登录功能，
    你觉得需要考虑哪些问题？"

AI：
1. 账号体系（邮箱/手机号/第三方登录）
2. 密码策略（强度、加密、找回）
3. Token 管理（存储、刷新、过期）
4. 安全性（防重放攻击、HTTPS）
5. 用户体验（自动登录、记住密码）

你："很好，按照这些要点实现"
```

## ✅ 小结

- Prompt 质量 = 输出质量
- 五要素：角色、任务、上下文、约束、示例
- 核心技巧：具体化、场景化、示例驱动、分步骤、提供反馈
- 避免常见错误：太简略、太模糊、缺上下文、期望过高、不反馈
