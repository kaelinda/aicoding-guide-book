# 01-结构化 Prompt

> Prompt 的质量决定输出质量。**结构 > 辞藻**。

---

## 为什么 Prompt 很重要

同一个需求，差的 Prompt 和好的 Prompt 结果差 10 倍。

```text
❌ "写个页面"
AI：[困惑] 什么页面？什么技术栈？
结果：生成的代码不符合预期，反复改

✅ "创建一个用户登录页面：
   - 邮箱输入框（带格式验证）
   - 密码输入框（至少 8 位）
   - 登录按钮（点击后调用 /auth/login）
   - 用 SwiftUI、MVVM"
AI：[理解需求] [按要求生成]
结果：一次到位
```

**AI 的输出质量 = 你的输入质量**。

---

## 五要素结构

```text
1. 角色（Role）
   "你是一个 iOS 开发专家..."

2. 任务（Task）
   "创建一个用户列表页面..."

3. 上下文（Context）
   "项目使用 SwiftUI + MVVM 架构..."

4. 约束（Constraints）
   "代码需要通过 SwiftLint 检查..."

5. 示例（Example）
   "参考这个格式：..."
```

### 完整示例

```text
【角色】你是有 10 年 iOS 经验的开发，精通 SwiftUI 和 MVVM。

【任务】创建一个商品列表页面。

【上下文】
- 项目：ShoppingApp
- 栈：Swift 5.9, SwiftUI, async/await（不用 Combine）
- 架构：MVVM

【约束】
- 遵循 Swift API 设计指南
- public 方法加 /// 文档注释
- UI 更新用 @MainActor
- 错误处理用 Result

【示例】
参考格式：
struct ProductListView: View {
    @State private var store: ProductListStore

    var body: some View {
        // 实现
    }
}
```

**经验**：前两要素（角色 + 任务）最低要求。**加上下文和约束之后效果开始稳定**。示例是锦上添花。

## 5 个核心技巧

### 技巧 1：具体化（用数字 / 名词替代形容词）

```text
❌ "优化一下性能"
✅ "优化商品列表页滚动性能：
   - 当前：滚动明显卡顿
   - 数据：约 100 个商品
   - 每项：图片（200x200）、标题、价格
   - 目标：60fps 流畅"
```

### 技巧 2：场景化（给出使用链路）

```text
❌ "写一个验证函数"
✅ "用户注册场景的邮箱验证：
   - 在注册页输入
   - 点击注册时验证
   - 失败显示红色提示
   - 通过才能提交表单"
```

### 技巧 3：示例驱动（给参考而不是凭空描述）

```text
❌ "写一个像微信的聊天界面"
✅ "参考微信聊天界面的布局：
   - 顶部：对方头像 + 昵称
   - 中间：消息列表（气泡）
     - 我：绿色气泡，右对齐
     - 对方：白色气泡，左对齐
   - 底部：输入框 + 发送按钮"
```

### 技巧 4：分步骤（避免一次吞象）

```text
❌ "开发一个完整的电商 App"
✅ "第一步：基础项目结构
   第二步：用户模块
   第三步：商品模块
   第四步：订单模块
   每步单独 Prompt，迭代完成。"
```

### 技巧 5：提供反馈（迭代优化）

```text
Round 1：你："创建用户列表页面"
        AI：[生成代码]

Round 2：你："样式不对，改卡片式布局，每卡片包含头像 + 用户名 + 简介"
        AI：[调整]

Round 3：你："好了，进下一个功能"
```

**别在 Round 1 直接 accept**。一次就想好所有细节不现实，迭代是常态。

## 质量自检清单

每次写完 Prompt，念一遍：

- [ ] 明确了**要做什么**？
- [ ] 说明了**为什么做**？
- [ ] 提供了**足够上下文**（技术栈 / 架构 / 相关文件）？
- [ ] 设了**约束条件**（风格 / 性能 / 兼容性）？
- [ ] 定义了**验收标准**（怎么算过）？
- [ ] 给了**示例或参考**（格式 / 风格）？
- [ ] 考虑了**边界情况**（空 / 错误 / 异常）？
- [ ] **结果可验证**（能跑测试？能看出对错？）

8 条中过 6 条以上，基本就是个好 Prompt。

---

## 质量等级

```text
Level 1 - 可用：
   一次基本完成，需要多次修改

Level 2 - 良好：
   一次完成 80%，少量调整

Level 3 - 优秀：
   一次完成 95%，符合所有要求

Level 4 - 完美：
   一次完成 100%，超出预期
```

**目标**：**Level 2-3 是常态**。追求 Level 4 是浪费时间（Prompt 写 20 分钟只为省 5 分钟改动）。
