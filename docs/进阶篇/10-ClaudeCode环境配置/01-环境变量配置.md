# Claude Code 环境变量配置

> 掌握 Claude Code 的核心配置：API Key、Base URL、代理、各种认证方式

## 1. API Key 配置

### 1.1 获取 API Key

Claude Code 支持以下账号类型：
- **Pro/Max/Team/Enterprise** — 直接使用 Claude 订阅
- **API Key** — 从 [console.claude.com](https://console.claude.com) 获取

> ⚠️ **注意**：免费版 Claude 不包含 Claude Code 使用权限。

### 1.2 设置 API Key

**方式 A：环境变量（推荐用于临时测试）**

```bash
# macOS / Linux
export ANTHROPIC_API_KEY=sk-ant-api03-xxxx

# Windows PowerShell
$env:ANTHROPIC_API_KEY="sk-ant-api03-xxxx"
```

**方式 B：写入配置文件（永久生效）**

```json
// ~/.claude/settings.json （用户级，所有项目生效）
{
  "env": {
    "ANTHROPIC_API_KEY": "sk-ant-api03-xxxx"
  }
}
```

```json
// .claude/settings.json （项目级，团队共享）
{
  "env": {
    "ANTHROPIC_API_KEY": "sk-ant-api03-xxxx"
  }
}
```

### 1.3 动态 API Key（企业场景）

如果你的 API Key 需要从密钥管理器动态获取，使用 `apiKeyHelper`：

```bash
#!/bin/bash
# ~/bin/get-api-key.sh

# 从 1Password 获取
# op run --no-session -- op item get "Claude API" --fields label=api-key

# 从 Vault 获取
vault kv get -field=api_key secret/claude/code

# 从环境变量（示例）
echo "${CLAUDE_API_KEY}"
```

```json
// settings.json
{
  "apiKeyHelper": "~/bin/get-api-key.sh"
}
```

> Key 会在每次会话开始时获取，可配合 TTL 设置刷新频率：
> ```json
> { "apiKeyHelperTTL": 3600000 }
> ```

### 1.4 多账号切换

不想混用 Key？创建多个 settings 文件：

```bash
# ~/.claude/settings.company.json
{
  "env": {
    "ANTHROPIC_API_KEY": "sk-ant-api03-公司Key"
  }
}

# ~/.claude/settings.personal.json
{
  "env": {
    "ANTHROPIC_API_KEY": "sk-ant-api03-个人Key"
  }
}
```

启动时指定配置：

```bash
claude --settings ~/.claude/settings.company.json
```

## 2. Base URL 配置

### 2.1 什么时候需要配置 Base URL

- 使用 **第三方代理**（如 LiteLLM、OpenRouter）
- 使用 **企业内网 API**
- 在 **国内** 需要走代理

### 2.2 标准 Anthropic API

默认不需要配置，直接用：

```bash
export ANTHROPIC_BASE_URL=https://api.anthropic.com
```

### 2.3 接入 LiteLLM 代理

```bash
# 统一端点（推荐）
export ANTHROPIC_BASE_URL=https://your-litellm-server:4000

# 或 provider-specific 端点
export ANTHROPIC_BASE_URL=https://your-litellm-server:4000/anthropic
```

settings.json 配置：

```json
{
  "env": {
    "ANTHROPIC_BASE_URL": "https://your-litellm-server:4000",
    "ANTHROPIC_AUTH_TOKEN": "sk-litellm-static-key"
  }
}
```

**LiteLLM 支持的 provider：**

| Provider | Endpoint |
|----------|----------|
| Anthropic | `/anthropic` |
| AWS Bedrock | `/bedrock` |
| Google Vertex | `/vertex_ai/v1` |

### 2.4 国内代理配置

如果在国内，需要设置 HTTP 代理：

```bash
# 环境变量
export HTTP_PROXY=http://127.0.0.1:7890
export HTTPS_PROXY=http://127.0.0.1:7890

# 或在 settings.json 中
{
  "env": {
    "HTTP_PROXY": "http://127.0.0.1:7890",
    "HTTPS_PROXY": "http://127.0.0.1:7890"
  }
}
```

## 3. 企业认证配置

### 3.1 AWS Bedrock

如果通过 AWS Bedrock 使用 Claude：

```bash
# 方式 1：环境变量
export ANTHROPIC_BEDROCK_BASE_URL=https://your-bedrock-endpoint
export AWS_ACCESS_KEY_ID=AKIA...
export AWS_SECRET_ACCESS_KEY=xxx
export AWS_REGION=us-east-1

# 方式 2：使用 AWS 默认凭证链（无需手动配置 Key）
# Claude Code 会自动读取 ~/.aws/credentials
```

settings.json：

```json
{
  "env": {
    "ANTHROPIC_BEDROCK_BASE_URL": "https://bedrock-runtime.us-east-1.amazonaws.com"
  }
}
```

> AWS Bedrock 支持 Entra ID 认证，需要配置 `AWS_PROFILE` 或使用 `aws sso login`。

### 3.2 Google Vertex AI

```bash
export ANTHROPIC_VERTEX_BASE_URL=https://us-central1-aiplatform.googleapis.com/v1
export ANTHROPIC_VERTEX_PROJECT_ID=your-gcp-project-id
export GOOGLE_APPLICATION_CREDENTIALS=/path/to/service-account.json
```

settings.json：

```json
{
  "env": {
    "ANTHROPIC_VERTEX_BASE_URL": "https://us-central1-aiplatform.googleapis.com/v1",
    "ANTHROPIC_VERTEX_PROJECT_ID": "your-gcp-project-id"
  }
}
```

### 3.3 Microsoft Azure Foundry

```bash
# 必需
export CLAUDE_CODE_USE_FOUNDRY=1
export ANTHROPIC_FOUNDRY_BASE_URL=https://your-resource.services.ai.azure.com/anthropic
export ANTHROPIC_FOUNDRY_API_KEY=your-api-key

# 可选：指定默认模型
export ANTHROPIC_DEFAULT_OPUS_MODEL=claude-opus-4-7
export ANTHROPIC_DEFAULT_SONNET_MODEL=claude-sonnet-4-6
export ANTHROPIC_DEFAULT_HAIKU_MODEL=claude-haiku-4-5
```

settings.json：

```json
{
  "env": {
    "CLAUDE_CODE_USE_FOUNDRY": "1",
    "ANTHROPIC_FOUNDRY_BASE_URL": "https://your-resource.services.ai.azure.com/anthropic",
    "ANTHROPIC_FOUNDRY_API_KEY": "your-api-key",
    "ANTHROPIC_DEFAULT_HAIKU_MODEL": "claude-haiku-4-5"
  }
}
```

> 也支持 Entra ID 认证（不设置 `ANTHROPIC_FOUNDRY_API_KEY` 时自动使用）。

### 3.4 自定义 OpenAI 兼容端点

Claude Code 原生不支持 OpenAI 格式，需要通过 **LLM 网关** 转换：

**方案 A：Claude Code Router（推荐）**

```bash
# 安装 ccr
git clone https://github.com/anthropics/claude-code-router.git
cd claude-code-router
npm install

# 配置 config.json
# 编辑 ~/claude-code-router/config.json

# 启动
./ccr
```

**方案 B：LiteLLM**

```bash
# 启动 LiteLLM
litellm --model ollama/llama2

# Claude Code 配置
export ANTHROPIC_BASE_URL=http://localhost:4000
```

| `ANTHROPIC_FOUNDRY_BASE_URL` | Foundry 端点 | `https://...` |
| `ANTHROPIC_FOUNDRY_API_KEY` | Foundry API Key | `xxx` |
| `HTTP_PROXY` / `HTTPS_PROXY` | HTTP 代理 | `http://127.0.0.1:7890` |
| `CLAUDE_CODE_USE_MANTLE` | 启用 Mantle 路由（v2.1.94+） | `1` |
| `CLAUDE_CODE_SANDBOXED` | 标记会话为沙箱/可信环境（v2.1.94+） | `1` |

| `CLAUDE_CODE_SUBPROCESS_ENV_SCRUB` | 从子进程中清除敏感环境变量（v2.1+） | `1` |
| `CLAUDE_CODE_CERT_STORE` | 证书存储策略：bundled/system/自定义路径（v2.1.101+） | `bundled` |
| `DISABLE_AUTOUPDATER` | 禁用自动更新 | `1` |
| `CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS` | 禁用实验功能 | `1` |

## 6. 常见问题

### Q1：设置了 API Key 但仍然报错 "Not authenticated"

检查：
1. Key 是否正确（注意 `sk-ant-api03` vs `sk-ant` 前缀）
2. 是否被代理拦截
3. 账号是否有 Claude Code 权限（需要 Pro/Max/Team/Enterprise）

### Q2：使用代理后连接不稳定

```bash
# 设置更长的超时
export ANTHROPIC_BASE_URL=https://api.anthropic.com
export HTTP_TIMEOUT=60
```

### Q3：多环境切换麻烦

使用 shell alias：

```bash
# ~/.zshrc
alias claude-work='ANTHROPIC_API_KEY=*** claude'
alias claude-home='ANTHROPIC_API_KEY=*** claude'
```

### Q4：企业安全配置（v2.1+ 新增）

**敏感环境变量清除** — 防止 API Key 泄露到子进程：

```bash
export CLAUDE_CODE_SUBPROCESS_ENV_SCRUB=1
```

这样 Bash 工具、hooks、MCP stdio servers 都会自动清除 `ANTHROPIC_*` 和云厂商相关的凭证。

**脚本限制** — 限制单个命令的 Bash 调用次数（防止异常循环）：

```json
{
  "env": {
    "CLAUDE_CODE_SCRIPT_CAPS": "{\"curl\":3,\"git\":5}"
  }
}
```

### Q5：国内证书问题（v2.1.101+ 新增）

如果遇到 SSL 证书错误，可以指定证书策略：

```bash
# 使用系统证书
export CLAUDE_CODE_CERT_STORE=system

# 使用自定义证书路径
export CLAUDE_CODE_CERT_STORE=/path/to/ca-bundle.crt
```

## ✅ 小结

1. **API Key**：优先通过 `settings.json` 配置，动态 Key 用 `apiKeyHelper`
2. **Base URL**：第三方代理或企业部署时需要配置
3. **代理**：国内通过 `HTTP_PROXY` / `HTTPS_PROXY` 环境变量
4. **企业认证**：Bedrock / Vertex / Foundry / Mantle 各有独立配置方式
5. **验证**：`claude doctor` 是排查问题的首选
6. **安全增强**（v2.1+）：使用 `CLAUDE_CODE_SUBPROCESS_ENV_SCRUB` 防止凭证泄露
