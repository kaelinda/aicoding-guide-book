# Claude Code 自定义模型配置

> 让 Claude Code 使用第三方模型、企业部署模型、自定义模型

## 1. 理解 Claude Code 的模型架构

Claude Code 原生使用 **Anthropic 模型**（Sonnet、Opus、Haiku）。

但通过配置，可以让 Claude Code 路由到：

| 类型 | 说明 | 配置难度 |
|------|------|----------|
| Anthropic 官方模型 | Pro/Max 订阅 | ⭐ 零配置 |
| AWS Bedrock | 企业级 Claude 部署 | ⭐⭐ |
| Google Vertex AI | GCP 上的 Claude | ⭐⭐ |
| Azure Foundry | Azure 上的 Claude | ⭐⭐ |
| LLM 网关（LiteLLM） | 统一管理多 provider | ⭐⭐⭐ |
| 自定义兼容端点 | OpenAI 兼容格式 | ⭐⭐⭐⭐ |

## 2. 模型别名系统

Claude Code 内置了一套模型别名，简化切换：

| 别名 | 实际模型 | 用途 |
|------|----------|------|
| `default` | 账户推荐模型 | 日常使用 |
| `best` | 最新最强模型 | 复杂任务 |
| `sonnet` | 最新 Sonnet | 日常编码 |
| `opus` | 最新 Opus | 复杂推理 |
| `haiku` | 最新 Haiku | 简单快速任务 |
| `sonnet[1m]` | 百万 token 上下文 | 超长代码分析 |
| `opus[1m]` | 百万 token 上下文 | 超长代码分析 |
| `opusplan` | Plan 用 Opus，执行用 Sonnet | 节省成本 |

### 切换模型

```bash
# 启动时指定
claude --model opus

# 会话中切换
/model sonnet
/model haiku

# 查看所有可用模型
/model
```

## 3. 模型参数配置

### 3.1 设置默认模型

```bash
# 环境变量
export ANTHROPIC_MODEL=claude-sonnet-4-6
```

```json
// settings.json
{
  "model": "claude-sonnet-4-6"
}
```

### 3.2 Effort Level（推理深度）

控制 Claude 的"思考深度"，影响输出质量和成本：

| Level | 适用场景 | Token 消耗 |
|--------|----------|------------|
| `low` | 简单、要求快速响应的任务 | 最少 |
| `medium` | 成本敏感的一般任务 | 适中 |
| `high` | **推荐默认值**，智能敏感工作 | 较高 |
| `xhigh` | **推荐用于编程**，最佳效果 | 高 |
| `max` | 高要求任务，但容易过度思考 | 最高 |

```bash
# 启动时设置
claude --effort xhigh

# 会话中调整
/effort high

# 环境变量
export CLAUDE_CODE_EFFORT_LEVEL=xhigh
```

settings.json：

```json
{
  "effortLevel": "xhigh"
}
```

> 建议：编程任务用 `xhigh`，简单修改用 `medium`。

### 3.3 自定义模型选项

通过环境变量添加自定义模型：

```bash
export ANTHROPIC_CUSTOM_MODEL_OPTION="my-gateway/claude-opus-4-7"
export ANTHROPIC_CUSTOM_MODEL_OPTION_NAME="Opus via Gateway"
export ANTHROPIC_CUSTOM_MODEL_OPTION_DESCRIPTION="通过内部 LLM 网关路由"
```

这样在 `/model` 列表中会显示为独立选项。

## 4. 限制可用模型

### 4.1 企业场景：限制模型列表

```json
// settings.json
{
  "availableModels": ["claude-sonnet-4-6", "claude-haiku-4-5"]
}
```

这样团队成员无法切换到昂贵的 Opus 模型。

### 4.2 模型 ID 映射

将标准模型映射到 provider 特定 ID：

```json
// settings.json
{
  "modelOverrides": {
    "claude-sonnet-4-6": "anthropic/claude-sonnet-4-6-custom",
    "claude-opus-4-7": "arn:aws:bedrock:us-east-1:123456789012:custo..."
  }
}
```

常见场景：
- 映射到 Bedrock 部署 ID
- 映射到 Vertex 部署名称
- 映射到 Foundry 部署名称

## 5. 第三方平台模型配置

### 5.1 AWS Bedrock

```bash
# 方式 1：直接配置
export ANTHROPIC_BEDROCK_BASE_URL=https://bedrock-runtime.us-east-1.amazonaws.com
export AWS_ACCESS_KEY_ID=AKIA...
export AWS_SECRET_ACCESS_KEY=...

# 方式 2：通过 LLM 网关
export ANTHROPIC_BASE_URL=https://your-litellm:4000/bedrock
```

**模型名称格式（Bedrock）：**

```bash
# Opus
export ANTHROPIC_DEFAULT_OPUS_MODEL=us.anthropic.claude-opus-4-7

# Sonnet
export ANTHROPIC_DEFAULT_SONNET_MODEL=us.anthropic.claude-sonnet-4-6

# Haiku
export ANTHROPIC_DEFAULT_HAIKU_MODEL=us.anthropic.claude-haiku-4-5
```

### 5.2 Google Vertex AI

```bash
export ANTHROPIC_VERTEX_BASE_URL=https://us-central1-aiplatform.googleapis.com/v1
export ANTHROPIC_VERTEX_PROJECT_ID=your-gcp-project-id
```

**模型名称格式（Vertex）：**

```bash
export ANTHROPIC_DEFAULT_OPUS_MODEL=claude-opus-4-7
export ANTHROPIC_DEFAULT_SONNET_MODEL=claude-sonnet-4-6
export ANTHROPIC_DEFAULT_HAIKU_MODEL=claude-haiku-4-5
```

### 5.3 Microsoft Azure Foundry

```bash
export CLAUDE_CODE_USE_FOUNDRY=1
export ANTHROPIC_FOUNDRY_BASE_URL=https://your-resource.services.ai.azure.com/anthropic
export ANTHROPIC_FOUNDRY_API_KEY=xxx
```

**模型名称格式（Foundry）：**

必须使用 Foundry portal 中配置的部署名称：

```bash
# 查看你的 Foundry 部署名称
# 假设部署名为 claude-sonnet-4-6
export ANTHROPIC_DEFAULT_SONNET_MODEL=claude-sonnet-4-6
export ANTHROPIC_DEFAULT_HAIKU_MODEL=claude-haiku-4-5
```

## 6. 通过 LLM 网关接入 Ollama / 本地模型

Claude Code 不原生支持 OpenAI 兼容端点，需要通过 **LLM 网关** 转换。

### 6.1 Claude Code Router（CCR）

专门为 Claude Code 设计的路由工具：

```bash
# 安装
git clone https://github.com/anthropics/claude-code-router.git
cd claude-code-router
npm install

# 配置
# 编辑 ~/claude-code-router/config.json
```

config.json 示例：

```json
{
  "LOG": true,
  "LOG_LEVEL": "debug",
  "HOST": "127.0.0.1",
  "PORT": 8080,
  "APIKEY": "",
  "PROXY_URL": "",
  "Providers": [
    {
      "name": "ollama",
      "api_base_url": "http://localhost:11434",
      "api_key": "",
      "models": ["llama3", "codellama"]
    }
  ],
  "Router": {
    "default": "ollama/llama3",
    "background": "ollama/llama3",
    "think": "ollama/llama3"
  }
}
```

启动：

```bash
./ccr
```

然后配置 Claude Code：

```bash
export ANTHROPIC_BASE_URL=http://127.0.0.1:8080
export ANTHROPIC_API_KEY=any-key  # CCR 不验证
```

### 6.2 LiteLLM

```bash
# 启动 LiteLLM 代理
litellm --model ollama/llama3 --port 4000

# Claude Code 配置
export ANTHROPIC_BASE_URL=http://localhost:4000
```

## 7. 高级配置：模型能力声明

如果使用第三方部署，需要声明模型支持的能力：

```bash
export ANTHROPIC_DEFAULT_OPUS_MODEL=your-model-id
export ANTHROPIC_DEFAULT_OPUS_MODEL_NAME="My Opus"
export ANTHROPIC_DEFAULT_OPUS_MODEL_DESCRIPTION="企业内网部署的 Opus"
export ANTHROPIC_DEFAULT_OPUS_MODEL_SUPPORTED_CAPABILITIES=effort,thinking
```

**支持的能力：**

| 能力 | 说明 |
|------|------|
| `effort` | 支持 effort level |
| `xhigh_effort` | 支持 xhigh 级别 |
| `max_effort` | 支持 max 级别 |
| `thinking` | 支持扩展思考 |
| `adaptive_thinking` | 支持自适应思考 |
| `interleaved_thinking` | 支持工具调用间思考 |

## 8. 百万 Token 上下文（1M 模型）

如果需要分析超长代码：

```bash
# 启用百万上下文
/model opus[1m]
/model sonnet[1m]

# 环境变量
export ANTHROPIC_DEFAULT_OPUS_MODEL='claude-opus-4-7[1m]'

# 禁用（节省成本）
export CLAUDE_CODE_DISABLE_1M_CONTEXT=1
```

**资格：**

| 计划 | Opus 1M | Sonnet 1M |
|------|---------|-----------|
| Max/Team/Enterprise | 免费 | 需额外付费 |
| Pro | 需额外付费 | 需额外付费 |
| API | 完整访问 | 完整访问 |

## 9. 常见问题

### Q1：第三方模型响应格式不对

Claude Code 依赖特定的响应格式。确保你的网关支持：

```
/v1/messages （必需）
/v1/messages/count_tokens （可选）
anthropic-beta: paddle1039-0-0 （如需要流式）
anthropic-version: 2023-06-01
```

### Q2：模型支持的能力不匹配

如果你禁用了某些能力（如 `thinking`），Claude Code 的某些功能可能不可用。

### Q3：如何确认当前用的哪个模型

```bash
# 查看状态
claude --status

# 会话中
/model
```

## ✅ 小结

1. **模型别名**：`/model` 命令是切换模型最简单的方式
2. **Effort Level**：`xhigh` 是编程推荐的默认值
3. **第三方平台**：Bedrock / Vertex / Foundry 各有独立配置
4. **本地模型**：需要 LLM 网关（CCR 或 LiteLLM）
5. **1M 上下文**：超长代码分析场景使用，注意成本
