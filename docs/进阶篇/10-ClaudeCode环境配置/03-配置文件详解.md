# Claude Code 配置文件详解

> settings.json 的完整字段说明和使用场景

## 1. 配置文件作用域

Claude Code 有 4 级配置文件，从高到低：

| 作用域 | 路径 | 影响范围 | 共享方式 |
|--------|------|----------|----------|
| **Managed** | `/etc/claude-code/managed-settings.json` | 全机器所有用户 | IT 批量部署 |
| **User** | `~/.claude/settings.json` | 你，跨所有项目 | 否 |
| **Project** | `.claude/settings.json` | 项目所有协作者 | 是（提交到 git） |
| **Local** | `.claude/settings.local.json` | 仅你在此项目 | 否（gitignored） |

**优先级（从高到低）：**

```
Local > Project > User > Managed
```

### 配置文件位置速查

| 功能 | User | Project | Local |
|------|------|---------|-------|
| Settings | `~/.claude/settings.json` | `.claude/settings.json` | `.claude/settings.local.json` |
| Subagents | `~/.claude/agents/` | `.claude/agents/` | — |
| MCP servers | `~/.mcp.json` | `.mcp.json` | `~/.mcp.json` |
| CLAUDE.md | `~/.claude/CLAUDE.md` | `CLAUDE.md` | `CLAUDE.local.md` |

## 2. 完整字段说明

### 2.1 核心配置

```json
{
  "model": "claude-sonnet-4-6",
  "effortLevel": "xhigh",
  "language": "chinese",
  "autoUpdatesChannel": "stable"
}
```

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `model` | string | 账户推荐 | 默认使用的模型 |
| `effortLevel` | string | `medium` | 推理深度 |
| `language` | string | 自动检测 | 响应语言（`chinese`、`japanese` 等） |
| `autoUpdatesChannel` | string | `latest` | 更新通道：`latest` 或 `stable` |

### 2.2 环境变量

```json
{
  "env": {
    "ANTHROPIC_API_KEY": "sk-ant-xxx",
    "NODE_ENV": "development",
    "MY_API_URL": "https://api.example.com"
  }
}
```

注入会话级环境变量，所有 Bash 命令和 hooks 都可以读取。

**典型用途：**
- 项目固定的 API 地址
- 开发/生产环境切换
- 敏感配置（配合 `.gitignore`）

### 2.3 权限配置

```json
{
  "permissions": {
    "defaultMode": "acceptEdits",
    "allow": [
      "Bash(npm run lint)",
      "Bash(npm run test)"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Read(./.env)",
      "Read(./secrets/**)"
    ]
  }
}
```

**权限模式：**

| 模式 | 行为 |
|------|------|
| `default` | 询问所有权限 |
| `acceptEdits` | 自动接受编辑 |
| `plan` | 自动接受阅读，拒绝修改 |
| `auto` | 基于规则自动决定 |
| `bypassPermissions` | 跳过所有确认（危险） |

**权限规则语法：**

```
ToolName(pattern)
Bash(curl *)
Read(.env)
Write(/tmp/test)
WebFetch(*.google.com)
```

### 2.4 MCP 服务器

```json
{
  "enableAllProjectMcpServers": true,
  "enabledMcpServers": ["github", "memory"],
  "disabledMcpServers": ["filesystem"]
}
```

| 字段 | 说明 |
|------|------|
| `enableAllProjectMcpServers` | 自动启用项目级 MCP |
| `enabledMcpServers` | 明确启用的 MCP 列表 |
| `disabledMcpServers` | 明确禁用的 MCP 列表 |

### 2.5 自动模式

```json
{
  "autoMode": {
    "environment": ["Trusted repo:*"]
  },
  "disableAutoMode": "disable",
  "alwaysThinkingEnabled": true
}
```

| 字段 | 说明 |
|------|------|
| `autoMode` | 自动模式规则 |
| `disableAutoMode` | 禁用自动模式 |
| `alwaysThinkingEnabled` | 默认启用扩展思考 |

### 2.6 Git 和提交

```json
{
  "attribution": {
    "coAuthoredBy": true,
    "coAuthorName": "Claude",
    "coAuthorEmail": "claude@example.com"
  },
  "includeGitInstructions": false
}
```

| 字段 | 说明 |
|------|------|
| `attribution.coAuthoredBy` | 是否添加 Co-authored-by |
| `attribution.coAuthorName` | AI 合著作者名称 |
| `attribution.coAuthorEmail` | AI 合著作者邮箱 |
| `includeGitInstructions` | 是否包含内置 Git 指令 |

### 2.7 Sandbox（沙箱）

```json
{
  "sandbox": {
    "enabled": true,
    "autoAllowBashIfSandboxed": true,
    "excludedCommands": ["docker *"],
    "filesystem": {
      "allowWrite": ["/tmp/build"],
      "denyRead": ["~/.aws/credentials"]
    },
    "network": {
      "allowedDomains": ["github.com", "*.npmjs.org"],
      "allowUnixSockets": ["/var/run/docker.sock"],
      "allowLocalBinding": true
    }
  }
}
```

**路径前缀：**

| 前缀 | 含义 |
|------|------|
| `/` | 绝对路径 |
| `~/` | 主目录 |
| `./` | 项目根目录 |

### 2.8 Plugins

```json
{
  "enabledPlugins": {
    "code-formatter@team-tools": true,
    "deployment-tools@team-tools": false
  },
  "extraKnownMarketplaces": {
    "acme-tools": {
      "source": {
        "source": "github",
        "repo": "acme-corp/claude-plugins"
      }
    }
  }
}
```

| 字段 | 说明 |
|------|------|
| `enabledPlugins` | 启用/禁用特定插件 |
| `extraKnownMarketplaces` | 添加额外的插件市场 |

**Marketplace 源类型：**

```json
// GitHub
{ "source": "github", "repo": "org/repo" }

// NPM
{ "source": "npm", "package": "@acme/claude-plugins" }

// 远程 URL
{ "source": "url", "url": "https://example.com/marketplace.json" }

// 本地文件
{ "source": "file", "path": "/path/to/marketplace.json" }
```

### 2.9 归属设置

```json
{
  "attribution": {
    "commit": "🤖 Generated with Claude Code",
    "pr": ""
  }
}
```

自定义提交和 PR 的归属信息。

### 2.10 其他设置

```json
{
  "cleanupPeriodDays": 60,
  "syntaxHighlightingDisabled": false,
  "terminalTitleFromRename": true,
  "spinnerTipsEnabled": true,
  "maxConcurrentMcpServers": 10,
  "toolUseDelayMs": 0,
  "compactOnSave": true,
  "alwaysThinkingEnabled": true
}
```

|| 字段 | 默认值 | 说明 |
||------|--------|------|
| `cleanupPeriodDays` | 30 | 会话保留天数，0 = 禁用持久化 |
| `syntaxHighlightingDisabled` | false | 禁用差异视图语法高亮 |
| `terminalTitleFromRename` | true | 允许 `/rename` 更新终端标题 |
| `spinnerTipsEnabled` | true | 显示思考提示 |
| `maxConcurrentMcpServers` | 10 | MCP 服务器最大并发数（v2.1+） |
| `toolUseDelayMs` | 0 | 工具调用间隔延迟（v2.1+） |
| `compactOnSave` | true | 保存时自动压缩上下文（v2.1+） |
| `alwaysThinkingEnabled` | false | 默认启用扩展思考（v2.1+） |

## 3. 实际配置示例

```json
// ~/.claude/settings.json
{
  "model": "claude-sonnet-4-6",
  "effortLevel": "xhigh",

  "env": {

    "coAuthorName": "My Claude",
    "coAuthorEmail": "claude@me.com"

```

### 3.2 团队项目配置

```json
// .claude/settings.json
{
  "model": "claude-sonnet-4-6",
  "availableModels": ["claude-sonnet-4-6", "claude-haiku-4-5"],
  "permissions": {
    "defaultMode": "plan",
    "allow": ["Bash(npm run *)", "Bash(git *)"],
    "deny": ["Bash(rm -rf node_modules)"]
  },
  "includeGitInstructions": false
}
```

### 3.3 企业托管配置

```json
// /etc/claude-code/managed-settings.json
{
  "model": "claude-sonnet-4-6",
  "autoUpdatesChannel": "stable",
  "permissions": {
    "defaultMode": "auto",
    "deny": ["Bash(curl *)", "Bash(wget *)"]
  },
  "sandbox": {
    "enabled": true,
    "network": {
      "allowedDomains": ["github.com", "*.company.com"]
    }
  }
}
```

### 3.4 国内代理配置

```json
// ~/.claude/settings.json
{
  "env": {
    "ANTHROPIC_API_KEY": "sk-ant-xxx",
    "HTTP_PROXY": "http://127.0.0.1:7890",
    "HTTPS_PROXY": "http://127.0.0.1:7890"
  }
}
```

### 3.5 多环境切换

```json
// .claude/settings.local.json （开发）
{
  "env": {
    "API_BASE_URL": "https://dev.api.com"
  }
}
```

```json
// .claude/settings.local.json （生产，手动切换）
{
  "env": {
    "API_BASE_URL": "https://api.company.com"
  }
}
```

## 4. 调试配置

### 4.1 查看当前配置

```bash
claude --status
```

### 4.2 配置诊断

```bash
claude doctor
```

会检查：
- API 连接
- 认证状态
- 配置完整性
- 环境变量

### 4.3 临时覆盖配置

```bash
# 启动时指定配置文件
claude --settings /path/to/custom-settings.json

# 指定模型
claude --model opus

# 指定工作目录
claude --workdir /path/to/project
```

## 5. 常见问题

### Q1：配置不生效？

检查优先级：
1. Local settings（`.claude/settings.local.json`）是否覆盖了 Project settings
2. 是否有环境变量 `ANTHROPIC_API_KEY` 覆盖了配置文件

### Q2：团队配置如何共享？

将 `.claude/settings.json` 提交到 git：
```bash
git add .claude/settings.json
git commit -m "chore: add Claude Code team config"
```

### Q3：敏感配置如何处理？

**不要**把 API Key 提交到 git：

```bash
# .gitignore
.claude/settings.local.json

# .claude/settings.json 只放非敏感配置
{
  "model": "claude-sonnet-4-6",
  "effortLevel": "xhigh"
}
```

## ✅ 小结

1. **作用域**：Local > Project > User > Managed
2. **核心字段**：`model`、`effortLevel`、`permissions`、`env`
3. **团队配置**：提交 `.claude/settings.json` 到 git
4. **敏感信息**：放在 `settings.local.json` 并加入 `.gitignore`
5. **调试**：`claude doctor` 是排查问题的首选
6. **新增字段**（v2.1+）：`maxConcurrentMcpServers`、`toolUseDelayMs`、`compactOnSave`、`alwaysThinkingEnabled`
