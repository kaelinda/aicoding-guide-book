## 什么是 Custom Roles

Custom Roles 是 Cursor 3.0（2026 年初）引入的全新功能，允许用户定义专属的 AI 行为角色。与默认的 AI 助手不同，自定义角色拥有特定的职责、语气和专业领域。你可以把它理解为"给 AI 换上不同的专业人设"——比如让它扮演资深 iOS 架构师、安全审计员，或者代码审查专家。

角色定义包含三个核心要素：**系统提示词**（决定 AI 的行为模式）、**快捷指令集**（预设的常用命令）和**上下文偏好**（默认包含哪些文件或规则）。三者组合让 AI 在特定场景下表现得更加专业和精准。

## 如何创建角色

### 基础创建流程

1. 打开 Cursor Settings → Features → Custom Roles
2. 点击 **+ New Role** 新建角色
3. 填写以下内容：

| 字段 | 说明 | 示例 |
|------|------|------|
| Name | 角色名称 | iOS 架构师 |
| Description | 简短描述 | 专注 Swift 架构设计与性能优化 |
| System Prompt | 行为定义 | 你是一位有 10 年经验的 iOS 架构师... |
| Shortcuts | 快捷指令 | `/review`、`/arch`、`/perf` |

### 系统提示词示例

```
你是一位专注于 Swift/SwiftUI 的高级 iOS 架构师。

工作原则：
- 优先推荐原生 Apple 方案
- 注重代码可维护性和扩展性
- 性能问题优先于功能堆砌

回复风格：
- 简洁直接，给出明确结论
- 附上关键代码片段
- 标注潜在风险点
```

### 快捷指令配置

快捷指令让角色在特定场景下快速激活：

```json
{
  "shortcuts": [
    {
      "trigger": "/review",
      "prompt": "审查当前代码，重点检查架构合理性、性能隐患和安全问题，输出结构化报告。"
    },
    {
      "trigger": "/arch",
      "prompt": "分析现有代码结构，识别模块边界，提出重构建议。"
    }
  ]
}
```

## 如何分享和使用角色

### 分享角色

- **导出**：在角色详情页点击 Export，生成 `.cursor-role.json` 文件
- **分享方式**：上传到 GitHub、Discord 或直接发送文件
- **隐私设置**：可选择公开（社区可见）或私有（仅自己使用）

### 导入他人角色

1. 获取 `.cursor-role.json` 文件
2. 在 Settings → Custom Roles → Import 导入
3. 选择角色并设为默认或按需切换

Cursor 官方也在建设 Role Marketplaces，未来可直接在编辑器内搜索和安装社区角色。

## 使用场景

### 场景一：专业模式切换

开发 iOS 应用时，可能同时需要：
- **Swift 架构师角色**：设计代码结构、定义模块边界
- **安全审计员角色**：检查敏感数据处理、API 调用安全
- **测试工程师角色**：生成单元测试、集成测试用例

通过快捷键 `Cmd/Ctrl + K` 快速切换角色，一个对话窗口内即可调用多个专家。

### 场景二：团队角色标准化

团队可统一创建"代码规范监督员"角色：
- 强制要求所有 AI 回复遵循团队规范
- 内置 lint 规则和命名约定
- 新成员导入角色后即可获得统一的 AI 辅助标准

### 场景三：面试/教学场景

创建"面试官"或"讲师"角色：
- 面试官角色：模拟算法题、技术八股提问
- 讲师角色：用通俗语言解释复杂概念，逐步引导

## 注意事项

- **提示词长度**：系统提示词建议控制在 500-1000 词，过长可能导致 AI 遗忘核心指令
- **角色冲突**：多个角色同时激活可能产生行为不一致，建议按需切换而非同时启用
- **版本兼容**：Custom Roles 功能需 Cursor 3.0 及以上版本，旧版本用户需升级
- **隐私风险**：导入他人角色时，注意检查其中是否包含恶意指令，建议在非关键项目先行测试
