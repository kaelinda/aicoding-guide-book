# 01-Agent 概念

> AI Agent = AI + 自主决策 + 工具使用

## 🤔 什么是 AI Agent

### 从 ChatBot 到 Agent

```
传统 ChatBot：
你：帮我写一个冒泡排序
AI：这是冒泡排序的代码...
你：[复制代码] → [打开编辑器] → [粘贴] → [保存] → [运行测试]

AI Agent：
你：帮我写一个冒泡排序
AI：[读取项目结构] → [创建文件] → [编写代码] → [运行测试] → [修复问题]
你：✅ 完成
```

**Agent 的核心特征：**

1. **自主性** - 能自己决定下一步做什么
2. **工具使用** - 能调用各种工具（读文件、写文件、运行命令）
3. **迭代改进** - 能根据结果调整策略
4. **环境感知** - 能理解当前上下文

### 形象理解

把 AI Agent 想象成一个**初级程序员**：

| 能力 | 说明 |
|------|------|
| 看得懂代码 | 可以读取并理解项目代码 |
| 会写代码 | 可以编写符合规范的代码 |
| 能执行命令 | 可以运行测试、构建项目 |
| 会调试 | 可以根据错误信息修复问题 |
| 需要指导 | 你给出需求，它负责实现 |

### 1. LLM 大脑

负责：
- 理解用户需求
- 规划执行步骤
- 决定使用什么工具
- 解析工具返回结果

主流 LLM：
- Claude（推荐用于编程）
- GPT-4 / GPT-5
- Gemini

### 2. 记忆系统

#### 短期记忆（对话上下文）
```
用户：创建一个计数器组件
AI：好的，我来创建... [执行中]
用户：再加个重置功能
AI：[记住之前的对话] 在刚才的组件里添加重置按钮...
```

#### 长期记忆（项目知识）
```
AGENTS.md：
- 项目使用 TypeScript
- 使用 React 18
- 样式用 Tailwind

AI 会记住这些约定，每次都遵守。
```

### 3. 工具箱

常见工具：

| 工具类别 | 具体功能 |
|---------|---------|
| 文件操作 | 读、写、删除、移动文件 |
| 代码分析 | 搜索、跳转定义、查找引用 |
| 执行命令 | 运行测试、构建、部署 |
| 网络请求 | 搜索文档、调用 API |
| Git 操作 | 提交、推送、创建分支 |

## 🎯 Agent 工作流程

### 示例：修复一个 Bug

```
1. 接收任务
   用户："修复登录页面的验证问题"
   
2. 理解需求
   AI："我需要找到登录页面的代码，理解验证逻辑"
   
3. 探索代码
   [工具：搜索文件] → 找到 LoginViewController.swift
   [工具：读文件] → 理解代码逻辑
   
4. 定位问题
   AI："发现问题：邮箱验证的正则表达式有误"
   
5. 实施修复
   [工具：编辑文件] → 修正正则表达式
   
6. 验证修复
   [工具：运行测试] → 测试通过
   
7. 汇报结果
   AI："已修复，问题在于正则表达式缺少转义字符"
```

### Agent vs 传统工具对比

| 场景 | 传统工具 | AI Agent |
|------|---------|----------|
| 重构代码 | 手动查找 → 手动修改 → 手动测试 | 描述需求 → Agent 自动完成 |
| 修复 Bug | 手动定位 → 手动修复 → 手动验证 | 描述 Bug → Agent 自主修复 |
| 写新功能 | 手动编写 → 手动调试 | 描述功能 → Agent 实现 |
| 代码审查 | 人工 Review | Agent 自动审查并修复 |

## 🤝 如何与 Agent 协作

### 正确的打开方式

#### ❌ 错误示范

```
你：写代码
AI：写什么代码？
你：随便
AI：[生成的代码不符合预期]
```

#### ✅ 正确示范

```
你：创建一个用户注册页面，包含：
    - 邮箱输入框（带验证）
    - 密码输入框（要求至少8位，包含数字和字母）
    - 确认密码输入框
    - 注册按钮
    使用 SwiftUI 实现
    
AI：[生成符合要求的代码]
```

### 给 Agent 的指令要素

```
1. 做什么（What）
   - 明确目标："创建一个登录页面"
   
2. 为什么做（Why，可选）
   - 帮助 AI 理解上下文："用于用户首次登录"
   
3. 怎么做（How，可选）
   - 技术约束："使用 SwiftUI"
   - 代码规范："遵循 MVVM 架构"
   
4. 验收标准（Done）
   - 什么时候算完成："测试通过，能正常登录"
```

### 高效协作技巧

#### 1. 提供足够的上下文

```
❌ "修一下这个 Bug"
✅ "LoginViewController.swift 第 45 行的邮箱验证正则表达式有问题，
    不允许包含 . 的邮箱（如 test.name@example.com），请修复"
```

#### 2. 明确优先级

```
"先实现基本功能，不用考虑性能优化"
"代码质量优先，时间不限"
"快速实现原型，后续会重构"
```

#### 3. 分步骤复杂任务

```
❌ "做一个完整的电商 App"
✅ "第一步：先实现商品列表页面"
   "第二步：添加商品详情页"
   "第三步：实现购物车功能"
```

#### 4. 及时反馈

```
AI：[生成了代码]
你：样式不太对，改用现代风格的设计
AI：[调整样式]
你：这个可以了，继续下一个功能
```

## 🛡 Agent 的边界

### Agent 擅长的

- ✅ 重复性工作（批量重命名、格式化代码）
- ✅ 规律性任务（写 CRUD 接口、创建组件）
- ✅ 信息检索（搜索代码、查找文档）
- ✅ 代码审查（发现潜在问题）
- ✅ 测试生成（写单元测试）

### Agent 不擅长的

- ❌ 创意设计（UI/UX 设计决策）
- ❌ 架构决策（技术选型、系统设计）
- ❌ 业务判断（需求优先级、用户价值）
- ❌ 处理模糊需求（需求本身不清晰时）

### 需要人工介入的场景

```
1. 关键决策
   - 选择技术栈
   - 确定架构方案
   - 处理安全敏感操作
   
2. 创意工作
   - UI/UX 设计
   - 品牌相关文案
   - 创新性功能设计
   
3. 质量把控
   - 代码审查最终确认
   - 上线前的最终测试
   - 性能优化方向决策
```

## 🎓 理解 Agent 的"思维"

### Agent 如何理解你的需求

```
你："添加一个搜索功能"

Agent 的思考过程：
1. 搜索功能？什么搜索？
   - 全局搜索？
   - 局部搜索？
   - 搜索什么内容？

2. 需要更多信息
   - 搜索哪些数据？（用户、商品、文章？）
   - 在哪个页面添加？
   - 需要实时搜索还是提交后搜索？

3. 可能的实现方式
   - 简单的字符串匹配
   - 全文检索（需要集成搜索引擎）
   - 调用后端 API

4. 最终选择
   → 倾向于最简单的实现
   → 如果不确定，会询问你
```

### 提升理解准确率的方法

1. **具体化** - "搜索商品列表，支持按名称模糊匹配"
2. **场景化** - "用户在首页输入关键词，实时显示匹配的商品"
3. **示例化** - "像淘宝首页的搜索框那样"

## 📊 Agent 的进化

```
Level 0: ChatBot
├─ 只能对话，不能执行操作

Level 1: Tool User
├─ 可以使用预设的工具
├─ 需要明确的指令

Level 2: Planner
├─ 可以拆解复杂任务
├─ 能规划执行步骤

Level 3: Self-Improver
├─ 能从错误中学习
├─ 可以自我反思和改进

Level 4: Multi-Agent Collaborator
├─ 多个 Agent 协作
├─ 分工明确，高效配合
```

**当前主流 Agent 在 Level 2-3 之间**

## ✅ 小结

- Agent = AI + 工具 + 自主决策
- Agent 有记忆、能执行、会迭代
- 与 Agent 协作需要：明确需求、提供上下文、及时反馈
- Agent 有边界，关键决策需要人工把关
