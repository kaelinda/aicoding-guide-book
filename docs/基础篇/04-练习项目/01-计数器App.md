# 01-计数器 App

> 用一个最小练习，掌握 AI 协作开发的基本节奏

## 🎯 项目目标

这是一个非常小，但非常适合入门的练习项目。

你需要完成的功能：

- 显示当前数字
- 点击按钮加 1
- 点击按钮减 1
- 点击按钮重置为 0

如果你是第一次正式用 AI 写一个小功能，建议从这里开始。

## 🤔 这个练习为什么重要

计数器看起来很简单，但它能帮你练到最关键的几个点：

- 如何清楚地描述一个功能
- 如何让 AI 生成最小可运行代码
- 如何继续增加功能，而不是每次推倒重来
- 如何自己验证结果是否正确

## 📋 需求说明

请用 SwiftUI 实现一个计数器页面，要求：

- 页面标题为“Counter Demo”
- 中间显示当前数字
- 提供 `+1`、`-1`、`重置` 三个按钮
- 布局保持简单清晰
- 代码尽量适合初学者理解

## 💬 推荐 Prompt

你可以把下面这段直接交给 AI：

```text
请帮我创建一个 SwiftUI 计数器页面，要求：
- 标题为 Counter Demo
- 中间显示当前数字
- 有 +1、-1、重置 三个按钮
- 使用最简单、适合初学者理解的写法
- 代码结构清晰
- 请解释每一部分代码的作用
```

## 🛠 推荐开发步骤

### 第一步：先生成最小版本
第一轮只要求 AI 做出：

- 数字显示
- 一个加法按钮

目的是先确保页面能跑。

### 第二步：再补减法和重置
当第一版运行正常后，再要求补充：

- `-1`
- `重置`

### 第三步：优化界面
功能没问题后，再让 AI 帮你调整：

- 按钮排版
- 间距
- 字体大小
- 圆角和背景

## ✅ 验收清单

完成后，用下面这份清单检查：

- [ ] 页面可以正常运行
- [ ] 点击 `+1` 后数字增加
- [ ] 点击 `-1` 后数字减少
- [ ] 点击 `重置` 后数字回到 0
- [ ] 页面布局清晰，没有明显错位
- [ ] 你能说清楚 `@State` 是干什么的

## ❌ 常见问题

### 问题 1：按钮点击了，但数字没变化
这通常意味着状态没有正确绑定。

你可以这样问 AI：

```text
按钮点击后数字没有更新。
请检查状态管理是否正确，并只做最小修改。
```

### 问题 2：AI 一次生成了太复杂的结构
如果 AI 上来就给你拆 ViewModel、拆多个文件，而你暂时还没准备好，可以这样收敛：

```text
请先用最简单的单文件 SwiftUI 写法实现，不要过度设计。
```

### 问题 3：我能运行，但还是不理解代码
你可以继续追问：

```text
请按初学者能理解的方式解释：
- @State 的作用
- Button 的 action 在哪里执行
- body 为什么会自动刷新
```

## 🚀 拓展练习

完成基础版后，可以继续加这些功能：

### 拓展 1：步长选择
增加 `+5`、`-5` 按钮，或者加一个步长选择器。

### 拓展 2：限制最小值
要求数字不能小于 0。

### 拓展 3：颜色反馈
当数字大于某个值时，文字颜色发生变化。

### 拓展 4：历史记录
记录最近几次点击结果。

## 💡 复盘建议

做完这个练习后，建议你自己回答下面几个问题：

1. 我给 AI 的需求够清楚吗？
2. 我有没有让 AI 一次输出太多？
3. 我是怎么发现 bug 的？
4. 我是否真的理解了生成结果？

把这些问题想清楚，比多做一个功能更重要。

## ✅ 小结

计数器 App 的重点不是功能本身，而是练习协作节奏：

- 小需求
- 小步迭代
- 每步都运行
- 出错就做最小修复

这就是 AI 编程最值得养成的基本习惯。
