## 📚 终端基础

### 什么是终端

**终端（Terminal）** 是与系统交互的命令行界面。

```
终端 = 仿真器 + Shell

仿真器：显示窗口，接收输入
Shell：解释命令，执行操作
```

### macOS 默认终端

```bash
# 默认 Shell
echo $SHELL
# /bin/zsh

# Shell 版本
zsh --version
# zsh 5.9

# 系统终端
# 应用 → 实用工具 → 终端.app
```

### 终端基础命令

```bash
# 文件操作
ls          # 列出文件
cd <dir>    # 进入目录
pwd         # 显示当前路径
mkdir <dir> # 创建目录
rm <file>   # 删除文件
cp <src> <dst>  # 复制文件
mv <src> <dst>  # 移动/重命名

# 查看文件
cat <file>  # 查看内容
less <file> # 分页查看
head <file> # 查看前几行
tail <file> # 查看后几行

# 搜索
find . -name "*.swift"  # 查找文件
grep "keyword" file.txt # 搜索内容

# 进程管理
ps aux      # 查看进程
kill <pid>  # 结束进程
top         # 系统监控

# 网络
ping google.com     # 测试连接
curl <url>          # HTTP 请求
ssh user@host       # SSH 连接
```

## 🛠 Shell 配置

### Zsh 配置文件

```bash
# Zsh 配置文件
~/.zshrc           # 用户配置（最常用）
~/.zprofile        # 登录 Shell 配置
~/.zshenv          # 环境变量
```

### 常用配置

```bash
# 编辑配置文件
nano ~/.zshrc

# 添加以下内容：

# ========== 别名 ==========
# Git
alias gs='git status'
alias ga='git add .'
alias gc='git commit -m'
alias gp='git push'
alias gl='git log --oneline'

# 导航
alias ..='cd ..'
alias ...='cd ../..'
alias ~='cd ~'

# 开发
alias run='npm run dev'
alias build='npm run build'
alias test='npm test'

# ========== 环境变量 ==========
# Node.js
export PATH="$(npm config get prefix)/bin:$PATH"

# nvm 延迟加载（加速启动）
export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && source "$NVM_DIR/nvm.sh" --no-use

# ========== 提示符 ==========
# 简洁提示符
PROMPT='%~ %# '

# 启用颜色
autoload -U colors && colors

# ========== 补全 ==========
# 启用补全
autoload -U compinit && compinit

# 补全时忽略大小写
zstyle ':completion:*' matcher-list 'm:{a-zA-Z}={A-Za-z}'

# ========== 历史 ==========
# 历史记录
HISTFILE=~/.zsh_history
HISTSIZE=10000
SAVEHIST=10000

# 不记录重复命令
setopt HIST_IGNORE_DUPS
setopt HIST_IGNORE_SPACE

# 保存配置后生效
source ~/.zshrc
```

### Oh My Zsh（可选）

```bash
# 安装 Oh My Zsh
sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"

# 主题配置
# 编辑 ~/.zshrc
ZSH_THEME="robbyrussell"  # 默认主题
ZSH_THEME="agnoster"      # 流行主题（需要 Powerline 字体）

# 插件配置
plugins=(git zsh-autosuggestions zsh-syntax-highlighting)

# 安装插件
git clone https://github.com/zsh-users/zsh-autosuggestions ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-autosuggestions
git clone https://github.com/zsh-users/zsh-syntax-highlighting ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-syntax-highlighting

# 生效配置
source ~/.zshrc
```

### 1. iTerm2

#### 简介
macOS 上最受欢迎的第三方终端，功能丰富。

#### 核心功能
- ✅ 分屏（水平/垂直）
- ✅ 热键窗口（快速唤起）
- ✅ 搜索功能（Cmd + F）
- ✅ 自动补全（Cmd + ;）
- ✅ 历史记录（Cmd + Shift + H）
- ✅ 触发器（自动执行命令）
- ✅ Shell Integration（Shell 集成）
- ✅ 密码管理器集成

#### 安装

```bash
# Homebrew 安装
brew install --cask iterm2

# 或直接下载
open https://iterm2.com/downloads.html
```

#### 配置建议

```yaml
Preferences → Appearance:
  - Theme: Minimal
  - Tab bar location: Top
  - Status bar: 启用

Preferences → Profiles:
  - Colors: Solarized Dark / Dracula
  - Font: JetBrains Mono (16pt)
  - Window: Transparency 10-15%

Preferences → Keys:
  - Hotkey: ⌥Space (快速唤起)
  - Key Mappings: 自定义快捷键
```

#### 常用快捷键

| 快捷键 | 功能 |
|--------|------|
| `Cmd + D` | 垂直分屏 |
| `Cmd + Shift + D` | 水平分屏 |
| `Cmd + ]` / `[` | 切换分屏 |
| `Cmd + Option + E` | 显示所有标签页 |
| `Cmd + Shift + H` | 历史记录 |
| `Cmd + ;` | 自动补全 |
| `Cmd + Shift + ;` | 命令历史 |

#### 优点
- ✅ 功能最全面
- ✅ 社区成熟，文档丰富
- ✅ 免费开源
- ✅ 高度可定制

#### 缺点
- ❌ 启动速度一般
- ❌ 内存占用较高
- ❌ 配置项太多，学习曲线陡

### 2. Ghostty

#### 简介
新一代终端仿真器，由 Mitchell Hashimoto（HashiCorp 创始人）开发，主打极速。

#### 核心特点
- ✅ **极速启动**：毫秒级启动
- ✅ **极致性能**：原生渲染，无卡顿
- ✅ **GPU 加速**：流畅滚动
- ✅ **低内存占用**：比 iTerm2 省 80%
- ✅ **现代设计**：简洁美观
- ✅ **原生体验**：macOS 原生设计

#### 安装

```bash
# Homebrew 安装
brew install --cask ghostty

# 或下载
open https://ghostty.org
```

#### 配置文件

```bash
# 创建配置文件
mkdir -p ~/.config/ghostty
nano ~/.config/ghostty/config

# 配置内容
font-family = "JetBrains Mono"
font-size = 16
theme = "dracula"
background-opacity = 0.95
window-padding-x = 10
window-padding-y = 10
cursor-style = block
cursor-style-blink = false

# 快捷键
keybind = cmd+d=new_split:right
keybind = cmd+shift+d=new_split:down
keybind = cmd+w=close_surface
```

#### 优点
- ✅ 启动速度最快
- ✅ 内存占用极低
- ✅ 性能卓越
- ✅ 配置简洁
- ✅ 持续更新

#### 缺点
- ❌ 功能相对较少
- ❌ 仍处于快速迭代期
- ❌ 社区较小

#### 适用场景
- 追求极致性能
- 简单终端使用
- 轻量级开发环境

### 3. tmux

#### 简介
终端复用器，在终端中创建多个会话和窗口。

#### 核心功能
- ✅ **会话管理**：断开后保持运行
- ✅ **窗口分屏**：灵活布局
- ✅ **远程持久**：SSH 断开后恢复
- ✅ **脚本化**：自动化布局

#### 安装

```bash
# Homebrew 安装
brew install tmux

# 验证
tmux -V
# tmux 3.4
```

#### 基础使用

```bash
# 启动新会话
tmux

# 创建命名会话
tmux new -s project

# 列出会话
tmux ls

# 连接会话
tmux attach -t project

# 断开会话（保持运行）
# 按 Ctrl+B，然后按 D

# 结束会话
tmux kill-session -t project
```

#### 配置文件

```bash
# 创建配置文件
nano ~/.tmux.conf

# 配置内容
# ========== 基础设置 ==========
set -g default-terminal "screen-256color"
set -g history-limit 50000
set -g base-index 1
setw -g pane-base-index 1

# ========== 快捷键 ==========
# 修改前缀键（默认 Ctrl+B）
set -g prefix C-a
unbind C-b

# 分屏
unbind '"'
bind - splitw -v -c "#{pane_current_path}"
unbind %
bind | splitw -h -c "#{pane_current_path}"

# 切换面板
bind h select-pane -L
bind j select-pane -D
bind k select-pane -U
bind l select-pane -R

# 快速重载配置
bind r source-file ~/.tmux.conf \; display "配置已重载"

# ========== 样式 ==========
# 状态栏
set -g status-bg black
set -g status-fg white
set -g status-interval 60
set -g status-left-length 30
set -g status-left '#[fg=green](#S) '
set -g status-right '#[fg=yellow]#(whoami)@#H #[fg=white]%H:%M'

# 面板边框
set -g pane-border-fg colour238
set -g pane-active-border-fg green

# ========== 鼠标支持 ==========
set -g mouse on
```

#### 常用快捷键

**前缀键**：`Ctrl+B`（或修改为 `Ctrl+A`）

| 快捷键 | 功能 |
|--------|------|
| `前缀 + ?` | 显示所有快捷键 |
| `前缀 + D` | 断开会话 |
| `前缀 + S` | 列出会话 |
| `前缀 + C` | 新建窗口 |
| `前缀 + N` / `P` | 下/上一个窗口 |
| `前缀 + 0-9` | 切换到指定窗口 |
| `前缀 + %` | 垂直分屏 |
| `前缀 + "` | 水平分屏 |
| `前缀 + 方向键` | 切换面板 |
| `前缀 + X` | 关闭当前面板 |

#### 使用场景

**场景 1：远程开发**
```bash
# SSH 到服务器
ssh user@server

# 启动 tmux
tmux new -s dev

# 开始工作...
# 即使断网，工作仍在后台运行

# 重新连接
ssh user@server
tmux attach -t dev
# 继续之前的工作！
```

**场景 2：多项目管理**
```bash
# 创建多个会话
tmux new -s frontend
tmux new -s backend
tmux new -s database

# 快速切换
tmux switch -t frontend
```

#### 优点
- ✅ SSH 持久化
- ✅ 分屏灵活
- ✅ 脚本自动化
- ✅ 跨平台

#### 缺点
- ❌ 学习曲线陡
- ❌ 快捷键复杂
- ❌ 需要额外配置

### 4. Tabby

#### 简介
现代化的跨平台终端，界面美观，功能丰富。

#### 核心功能
- ✅ 跨平台（Windows/macOS/Linux）
- ✅ SSH/SFTP 客户端
- ✅ 串口连接
- ✅ 同步配置（云同步）
- ✅ 自动补全
- ✅ 分屏
- ✅ 主题丰富

#### 安装

```bash
# Homebrew 安装
brew install --cask tabby

# 或下载
open https://eugeny.github.io/tabby/
```

#### 配置

```
设置 → 配置文件：
- Shell 配置：选择默认 Shell
- 外观：选择主题
- 字体：JetBrains Mono

设置 → SSH：
- 添加服务器连接
- 保存密码（可选）
- 配置密钥认证

设置 → 同步：
- GitHub 登录
- 同步配置到云端
```

#### 特点
- ✅ 界面现代化
- ✅ SSH 管理方便
- ✅ 跨平台同步
- ✅ 开箱即用

#### 优点
- ✅ 美观易用
- ✅ 内置 SSH 客户端
- ✅ 配置云同步
- ✅ 新手友好

#### 缺点
- ❌ 启动速度较慢（Electron）
- ❌ 内存占用较高
- ❌ 功能深度不如 iTerm2

## 🎯 工具选择指南

### 决策树

```
你用什么操作系统？

├─ macOS
│   ├─ 追求性能？
│   │   ├─ 是 → Ghostty（极速启动）
│   │   └─ 否 → iTerm2（功能最全）
│   └─ 需要远程持久化？
│       └─ 是 → iTerm2 + tmux
│
├─ Windows
│   └─ Tabby（现代化体验）
│
└─ Linux
    ├─ 简单使用 → Ghostty
    └─ 高级需求 → tmux
```

### 场景推荐

| 场景 | 推荐工具 | 理由 |
|------|----------|------|
| **新手入门** | Tabby | 界面友好，开箱即用 |
| **日常开发** | iTerm2 | 功能全面，社区成熟 |
| **追求性能** | Ghostty | 启动快，占用低 |
| **远程开发** | iTerm2 + tmux | 会话持久，不怕断网 |
| **跨平台** | Tabby | 配置同步，体验一致 |
| **极简主义** | Terminal.app + tmux | 系统自带，功能够用 |

### 组合推荐

**方案 1：性能优先**
```
Ghostty（终端） + 内置分屏
优点：极速、轻量
适合：简单开发场景
```

**方案 2：功能优先**
```
iTerm2（终端） + Oh My Zsh
优点：功能最强，生态丰富
适合：深度开发者
```

**方案 3：远程开发**
```
iTerm2（终端） + tmux（会话管理）
优点：SSH 持久化，不怕断网
适合：服务器开发、远程协作
```

**方案 4：跨平台**
```
Tabby（终端）
优点：配置云同步，体验一致
适合：多系统切换的用户
```

## 💡 提升效率的技巧

### 1. 快捷键效率

```bash
# 光标移动
Ctrl + A    # 移到行首
Ctrl + E    # 移到行尾
Ctrl + B    # 向左移动一个字符
Ctrl + F    # 向右移动一个字符
Alt + B     # 向左移动一个词
Alt + F     # 向右移动一个词

# 编辑
Ctrl + W    # 删除前一个词
Ctrl + U    # 删除到行首
Ctrl + K    # 删除到行尾
Ctrl + Y    # 粘贴删除的内容

# 历史
Ctrl + R    # 搜索历史命令
Ctrl + P    # 上一条命令
Ctrl + N    # 下一条命令
```

### 2. 别名提效

```bash
# 添加到 ~/.zshrc

# Git
alias gs='git status'
alias ga='git add .'
alias gc='git commit -m'
alias gp='git push'
alias gl='git log --oneline --graph'
alias gd='git diff'

# 导航
alias ..='cd ..'
alias ...='cd ../..'
alias ....='cd ../../..'
alias ~='cd ~'
alias -='cd -'

# 开发
alias run='npm run dev'
alias build='npm run build'
alias test='npm test'
alias lint='npm run lint'

# 系统
alias ls='ls -G'
alias ll='ls -lah'
alias la='ls -A'
alias cls='clear'
alias h='history'
alias j='jobs'

# 快速编辑
alias zshconfig='nano ~/.zshrc'
alias reload='source ~/.zshrc'
```

### 3. 函数提效

```bash
# 添加到 ~/.zshrc

# 创建目录并进入
mkcd() {
    mkdir -p "$1" && cd "$1"
}

# 快速查找文件
ff() {
    find . -type f -iname "*$1*"
}

# 快速查找目录
fd() {
    find . -type d -iname "*$1*"
}

# Git 提交并推送
gcp() {
    git add . && git commit -m "$1" && git push
}

# 快速创建项目
new-project() {
    mkdir -p "$1"
    cd "$1"
    git init
    touch README.md
    echo "# $1" > README.md
}
```

### 4. 自动跳转

```bash
# 安装 z（自动跳转工具）
brew install z

# 使用
cd ~/Projects/my-app
z my-app  # 自动跳转

# 安装 zoxide（更快的 z）
brew install zoxide

# 添加到 ~/.zshrc
eval "$(zoxide init zsh)"

# 使用
z my-app
```
