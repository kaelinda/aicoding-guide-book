# 05-环境验证

> 确保一切就绪，开始 AI 编程之旅

## 🎯 验证目标

完成本章后，你将确认：
- ✅ 所有工具正确安装
- ✅ 环境变量配置正确
- ✅ 版本符合要求
- ✅ 可以开始使用 AI 编程工具

## 📋 验证清单

### 步骤 1：验证 Homebrew

```bash
# 检查 Homebrew 版本
brew --version

# 预期输出：
# Homebrew 4.x.x
# Homebrew/homebrew-core (git revision xxx; last commit xxx)
```

**如果失败：** [回到 Homebrew 安装](./01-homebrew.md)

### 步骤 2：验证 nvm

```bash
# 检查 nvm 版本
nvm --version

# 预期输出：
# 0.39.7（或更高版本）
```

**如果失败：** [回到 nvm 安装](./02-nvm.md)

### 步骤 3：验证 Node.js

```bash
# 检查 Node.js 版本
node -v

# 预期输出：
# v20.11.0（或更高 LTS 版本）

# 检查 npm 版本
npm -v

# 预期输出：
# 10.2.4（或更高版本）
```

**要求：**
- Node.js 版本 ≥ 18
- 推荐：Node.js 20 LTS 或更高

**如果失败：** [回到 Node.js 配置](./03-nodejs.md)

### 步骤 4：验证 Git

```bash
# 检查 Git 版本
git --version

# 预期输出：
# git version 2.44.0（或更高版本）

# 检查 Git 配置
git config user.name
git config user.email

# 预期输出：
# Your Name
# your.email@example.com
```

**如果失败：** [回到 Git 配置](./04-git.md)

### 步骤 5：验证 SSH（可选但推荐）

```bash
# 测试 GitHub 连接
ssh -T git@github.com

# 预期输出：
# Hi username! You've successfully authenticated, but GitHub does not provide shell access.
```

**如果失败：** 重新配置 SSH 密钥

## 🛠 一键验证脚本

创建并运行验证脚本：

```bash
# 创建脚本
cat > ~/check-env.sh << 'EOF'
#!/bin/bash

echo "🔍 开始环境验证..."
echo ""

# 颜色定义
GREEN='\033[0;32m'
RED='\033[0;31m'
NC='\033[0m' # No Color

# 验证函数
check_command() {
    if command -v $1 &> /dev/null; then
        echo -e "${GREEN}✅ $1 已安装${NC}"
        $1 --version 2>&1 | head -1
        echo ""
        return 0
    else
        echo -e "${RED}❌ $1 未安装${NC}"
        echo ""
        return 1
    fi
}

# 检查 Homebrew
echo "📦 检查 Homebrew..."
check_command brew

# 检查 nvm
echo "📦 检查 nvm..."
if [ -s "$HOME/.nvm/nvm.sh" ]; then
    echo -e "${GREEN}✅ nvm 已安装${NC}"
    source "$HOME/.nvm/nvm.sh"
    nvm --version
    echo ""
else
    echo -e "${RED}❌ nvm 未安装${NC}"
    echo ""
fi

# 检查 Node.js
echo "📦 检查 Node.js..."
check_command node

# 检查 npm
echo "📦 检查 npm..."
check_command npm

# 检查 Git
echo "📦 检查 Git..."
check_command git

# 检查 Git 配置
echo "⚙️ 检查 Git 配置..."
if git config user.name &> /dev/null && git config user.email &> /dev/null; then
    echo -e "${GREEN}✅ Git 已配置${NC}"
    echo "用户名: $(git config user.name)"
    echo "邮箱: $(git config user.email)"
    echo ""
else
    echo -e "${RED}⚠️ Git 需要配置用户信息${NC}"
    echo "运行: git config --global user.name \"Your Name\""
    echo "运行: git config --global user.email \"your.email@example.com\""
    echo ""
fi

# 检查 SSH
echo "🔑 检查 SSH..."
if [ -f "$HOME/.ssh/id_ed25519.pub" ] || [ -f "$HOME/.ssh/id_rsa.pub" ]; then
    echo -e "${GREEN}✅ SSH 密钥已存在${NC}"
    echo ""
else
    echo -e "${RED}⚠️ SSH 密钥未配置${NC}"
    echo "运行: ssh-keygen -t ed25519 -C \"your.email@example.com\""
    echo ""
fi

echo "✨ 环境验证完成！"
EOF

# 添加执行权限
chmod +x ~/check-env.sh

# 运行脚本
~/check-env.sh
```

## ⚠️ 如果验证失败

### Homebrew 问题

```bash
# 重新安装
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 配置环境变量（M 系列）
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"
```

### nvm 问题

```bash
# 重新安装
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# 重新加载配置
source ~/.zshrc
```

### Node.js 问题

```bash
# 通过 nvm 安装
nvm install --lts
nvm use --lts
```

### Git 问题

```bash
# 通过 Homebrew 安装
brew install git

# 配置用户信息
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
```

### SSH 问题

```bash
# 生成密钥
ssh-keygen -t ed25519 -C "your.email@example.com"

# 启动 SSH agent
eval "$(ssh-agent -s)"

# 添加密钥
ssh-add ~/.ssh/id_ed25519
```
