# 04-Git 配置

> 版本控制必备，AI 编程的好搭档

## 🤔 为什么 Git 很重要

### AI 编程场景下的 Git

```
场景 1：AI 修改了代码，你想回退
→ Git 可以回退到任何历史版本

场景 2：AI 生成的代码有问题
→ Git diff 可以清楚看到改了什么

场景 3：多分支开发
→ Git 分支让 AI 工作互不干扰

场景 4：团队协作
→ Git 是团队协作的基础设施
```

**结论：** Git 是 AI 编程的安全网和协作工具。

## 📦 安装 Git

### macOS 自带 Git

```bash
# 检查是否已安装
git --version
# 输出：git version 2.39.3 (Apple Git-146)

# 如果未安装，会提示安装 Command Line Tools
```

### 通过 Homebrew 安装（推荐）

```bash
# 安装最新版 Git
brew install git

# 验证安装
git --version
# 输出：git version 2.44.0

# 查看安装路径
which git
# 输出：/opt/homebrew/bin/git
```

## 🔧 基础配置

### 用户信息

```bash
# 设置用户名
git config --global user.name "Your Name"

# 设置邮箱
git config --global user.email "your.email@example.com"

# 验证配置
git config --global user.name
git config --global user.email
```

### 编辑器设置

```bash
# 使用 VS Code 作为编辑器
git config --global core.editor "code --wait"

# 或使用 Vim
git config --global core.editor vim

# 或使用 Nano
git config --global core.editor nano
```

### 分支命名

```bash
# 设置默认分支名
git config --global init.defaultBranch main

# 验证
git config --global init.defaultBranch
```

### 查看所有配置

```bash
# 查看全局配置
git config --global --list

# 查看所有配置（包括系统、全局、本地）
git config --list
```

## 🔑 SSH 密钥配置

### 生成 SSH 密钥

```bash
# 生成新密钥（推荐 Ed25519）
ssh-keygen -t ed25519 -C "your.email@example.com"

# 或生成 RSA 密钥（兼容性更好）
ssh-keygen -t rsa -b 4096 -C "your.email@example.com"

# 按提示操作：
# 1. 选择保存路径（默认 ~/.ssh/id_ed25519）
# 2. 设置密码（可选，建议设置）
# 3. 确认密码
```

### 添加到 GitHub

```bash
# 1. 复制公钥
cat ~/.ssh/id_ed25519.pub
# 或
pbcopy < ~/.ssh/id_ed25519.pub  # macOS

# 2. 添加到 GitHub
# 访问：https://github.com/settings/keys
# 点击 "New SSH key"
# 粘贴公钥内容
# 保存

# 3. 测试连接
ssh -T git@github.com
# 输出：Hi username! You've successfully authenticated...
```

### 添加到多个平台

```bash
# GitLab
ssh -T git@gitlab.com

# Bitbucket
ssh -T git@bitbucket.org

# Gitee（国内）
ssh -T git@gitee.com
```

## 📝 .gitignore 配置

### 什么是 .gitignore

指定 Git 应该忽略的文件和目录，避免提交：
- 编译产物
- 依赖目录
- 敏感信息
- IDE 配置

## 🎯 常用命令

### 初始化与克隆

```bash
# 初始化新仓库
git init

# 克隆远程仓库
git clone git@github.com:user/repo.git

# 克隆指定分支
git clone -b branch-name git@github.com:user/repo.git

# 浅克隆（只克隆最近一次提交）
git clone --depth 1 git@github.com:user/repo.git
```

### 日常工作流

```bash
# 查看状态
git status

# 添加到暂存区
git add file.txt
git add .  # 添加所有修改

# 提交
git commit -m "feat: 添加新功能"

# 推送
git push origin main

# 拉取
git pull origin main
```

### 分支管理

```bash
# 查看分支
git branch          # 本地
git branch -r       # 远程
git branch -a       # 所有

# 创建分支
git branch new-feature

# 切换分支
git checkout new-feature

# 创建并切换
git checkout -b new-feature

# 删除分支
git branch -d old-feature

# 合并分支
git checkout main
git merge new-feature
```

### 查看历史

```bash
# 查看提交历史
git log
git log --oneline  # 简洁模式
git log --graph    # 图形化显示

# 查看文件历史
git log --follow file.txt

# 查看差异
git diff                    # 未暂存的修改
git diff --staged          # 已暂存的修改
git diff main..develop     # 分支差异
```

### 撤销操作

```bash
# 撤销工作区修改
git checkout -- file.txt

# 撤销暂存
git reset HEAD file.txt

# 撤销提交（保留修改）
git reset --soft HEAD~1

# 撤销提交（丢弃修改）
git reset --hard HEAD~1

# 修改最后一次提交
git commit --amend
```

## 🌟 高级技巧

### Git 别名

```bash
# 设置常用别名
git config --global alias.st status
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit
git config --global alias.unstage 'reset HEAD --'
git config --global alias.last 'log -1 HEAD'
git config --global alias.visual 'log --graph --oneline --all'

# 使用别名
git st
git co main
git visual
```

### Git Hooks

```bash
# pre-commit hook
# .git/hooks/pre-commit

#!/bin/sh
# 运行 SwiftLint
swiftlint

# 运行测试
xcodebuild test -scheme MyProject

# 如果失败，阻止提交
```

### Git Stash

```bash
# 暂存当前修改
git stash

# 查看暂存列表
git stash list

# 恢复暂存
git stash pop

# 恢复指定暂存
git stash apply stash@{0}
```

### Git Rebase

```bash
# 交互式 rebase
git rebase -i HEAD~3

# 变基到 main 分支
git checkout feature
git rebase main

# 中途冲突解决
git rebase --continue  # 继续
git rebase --abort     # 放弃
```

## ⚠️ 常见问题

### 问题 1：提交了敏感信息

```bash
# 从历史中删除文件
git filter-branch --force --index-filter \
  'git rm --cached --ignore-unmatch path/to/sensitive-file' \
  --prune-empty --tag-name-filter cat -- --all

# 强制推送（谨慎使用）
git push origin --force --all
```

### 问题 2：合并冲突

```bash
# 查看冲突文件
git status

# 手动解决冲突
# 编辑冲突文件，选择保留的内容

# 标记为已解决
git add conflict-file.txt

# 完成合并
git commit
```

### 问题 3：撤销已推送的提交

```bash
# 创建反向提交
git revert commit-hash

# 推送
git push origin main
```
