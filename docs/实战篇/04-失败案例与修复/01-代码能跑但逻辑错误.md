# 01-代码能跑但逻辑错误

> 代码能编译、能运行，但结果却不对。这种 bug 最难发现。

---

## 问题特征

- ✅ 编译通过
- ✅ 运行不崩溃
- ❌ 结果不符合预期
- ❌ 边界情况处理错误

---

## 案例一：计数器负数

### 问题描述

AI 生成的计数器允许数字变成负数，但我们要求"不能小于 0"。

### 错误代码

```swift
struct CounterView: View {
    @State private var count = 0
    
    var body: some View {
        VStack {
            Text("\(count)")
            
            Button("减少") {
                count -= 1  // 没有检查
            }
        }
    }
}
```

### 问题分析

```swift
// 问题：没有边界检查
count -= 1  // 可以变成 -1, -2, -3...

// AI 可能理解为"数学上正确"，而不是"业务逻辑正确"
```

### 修复代码

```swift
Button("减少") {
    if count > 0 {  // 添加边界检查
        count -= 1
    }
}
```

### 改进 Prompt

❌ 模糊描述：
```
"实现计数器，点击减少"
```

✅ 明确约束：
```
"实现计数器：
- 点击+，数字+1
- 点击-，数字-1
- 数字最小为 0，不能出现负数
- 如果已经是 0，点击-无反应"
```

## 案例二：搜索匹配错误

### 问题描述

AI 生成的搜索功能，大小写不敏感没生效。

### 错误代码

```swift
func search(query: String) -> [Product] {
    products.filter { product in
        product.name.contains(query)  // 大小写敏感！
    }
}
```

### 问题分析

```swift
// "iPhone" 可以被找到
// "iphone" 找不到
// "IPHONE" 找不到

.contains(query)  // Swift String 默认大小写敏感
```

### 修复代码

```swift
func search(query: String) -> [Product] {
    products.filter { product in
        product.name.localizedCaseInsensitiveContains(query)
    }
}
```

### 改进 Prompt

```
"实现商品搜索：
- 支持大小写不敏感匹配
- 'iphone' 能匹配到 'iPhone'
- 'IPHONE' 也能匹配到 'iPhone'"
```

## 案例三：异步数据竞态

### 问题描述

商品列表页显示的数据是旧的，API 返回后 UI 没更新。

### 错误代码

```swift
struct ProductListView: View {
    @State private var products: [Product] = []
    
    var body: some View {
        List(products) { product in
            ProductRow(product: product)
        }
        .onAppear {
            loadProducts()
        }
    }
    
    func loadProducts() {
        // 问题：异步请求，但函数没有标记 async
        API.getProducts { result in
            products = result  // 直接赋值，UI 不会更新
        }
    }
}
```

### 问题分析

```swift
// 问题1：闭包内修改 @State 变量
// 问题2：没有使用 Task 或 @MainActor
// 问题3：网络请求是异步的，但 UI 可能先显示了空列表
```

### 修复代码

```swift
struct ProductListView: View {
    @State private var products: [Product] = []
    @State private var isLoading = true
    
    var body: some View {
        Group {
            if isLoading {
                ProgressView("加载中...")
            } else {
                List(products) { product in
                    ProductRow(product: product)
                }
            }
        }
        .task {
            await loadProducts()
        }
    }
    
    func loadProducts() async {
        isLoading = true
        do {
            products = try await API.getProducts()
        } catch {
            print("加载失败: \(error)")
        }
        isLoading = false
    }
}
```

### 改进 Prompt

```
"实现商品列表：
- 页面显示加载指示器
- 网络请求使用 async/await
- 加载完成后显示商品列表
- 网络错误显示错误提示
- 使用 @State 管理加载状态"
```

## 案例四：数组越界

### 问题描述

用户反馈 App 偶尔会闪退，查日志是数组越界。

### 错误代码

```swift
func getNextItem(index: Int) -> Item? {
    return items[index + 1]  // 没有检查 bounds
}
```

### 问题分析

```swift
// 如果 index 是最后一个元素
// index + 1 = items.count  → 越界！

// 或者：
// items[index]  → 如果 index >= items.count，越界
```

### 修复代码

```swift
func getNextItem(index: Int) -> Item? {
    let nextIndex = index + 1
    guard nextIndex < items.count else { return nil }
    return items[nextIndex]
}

// 或者更简洁
func getNextItem(index: Int) -> Item? {
    guard index + 1 < items.count else { return nil }
    return items[index + 1]
}
```

### 改进 Prompt

```
"实现列表导航：
- 获取下一个元素前检查是否存在
- 如果没有下一个元素，返回 nil
- 不要使用 force unwrap (!)"
```

## 常见错误模式总结

| 错误模式 | 风险 | 修复方法 |
|----------|------|----------|
| 缺少边界检查 | 高 | 明确说明边界条件 |
| 大小写敏感 | 中 | 指定 case-insensitive |
| 异步状态更新 | 高 | 使用 @MainActor / Task |
| Force unwrap | 高 | 使用 guard / if let |
| 空数组处理 | 中 | 说明空状态显示 |
| 时间格式错误 | 低 | 指定具体格式 |

## 调试技巧

### 1. 打印日志

```swift
print("DEBUG: count = \(count)")
print("DEBUG: items.count = \(items.count)")
```

### 2. 断点调试

在 Xcode 中设置断点，观察变量值。

### 3. 单元测试

```swift
func testCounterCannotGoNegative() {
    var counter = Counter()
    counter.decrement()  // 从 0 减
    XCTAssertEqual(counter.value, 0)  // 应该是 0，不是 -1
}
```

```
□ 是否说明了边界条件？

□ 是否有性能要求？

□ 是否禁止使用 force unwrap？
```
