# 01 代码能跑但逻辑错误

> 最危险的错误：代码能跑，但结果不对

---

## 问题特点

- ✅ 代码没有语法错误
- ✅ 不会崩溃或报错
- ⚠️ 但计算/判断结果**是错的**
- ⚠️ 很难被发现

---

## 案例 1：条件判断反了

### 场景

用户权限检查：VIP 用户可以访问高级功能

### AI 生成的代码

```swift
func canAccessPremium(user: User) -> Bool {
    // 检查是否是 VIP
    if user.isVIP == false {
        return true  // ⚠️ 逻辑反了！
    }
    return false
}
```

### 问题

| 用户类型 | 预期结果 | 实际结果 |
|----------|----------|----------|
| VIP 用户 | ✅ true | ❌ false |
| 普通用户 | ❌ false | ✅ true |

### 正确代码

```swift
func canAccessPremium(user: User) -> Bool {
    if user.isVIP == true {  // 正确
        return true
    }
    return false
}

// 或更简洁
func canAccessPremium(user: User) -> Bool {
    return user.isVIP
}
```

### 怎么发现

```bash
# 测试用例
你：帮我写测试用例验证这个函数

# 测试
canAccessPremium(VIP用户) 应返回 true
canAccessPremium(普通用户) 应返回 false
```

## 案例 2：循环范围错误

### 场景

计算数组前 N 个元素的和

### AI 生成的代码

```swift
func sumFirstN(numbers: [Int], n: Int) -> Int {
    var sum = 0
    for i in 0...n {  // ⚠️ 包含 n，多算了
        sum += numbers[i]
    }
    return sum
}
```

### 问题

- `0...n` 包含 n，实际取了 n+1 个元素
- 数组越界风险（如果 n >= numbers.count）

### 正确代码

```swift
func sumFirstN(numbers: [Int], n: Int) -> Int {
    var sum = 0
    let count = min(n, numbers.count)  // 边界保护
    for i in 0..<count {  // 半开区间，不包含 count
        sum += numbers[i]
    }
    return sum
}
```

### 怎么发现

```swift
// 测试
sumFirstN([1,2,3,4,5], n: 2)
// 预期: 1 + 2 = 3
// 实际: 1 + 2 + 3 = 6 ❌
```

## 案例 3：异步回调处理错误

### 场景

用户登录后获取用户信息

### AI 生成的代码

```swift
func loginAndGetUser() -> User? {
    var user: User?
    login { success in
        if success {
            user = getUser()  // ⚠️ 闭包里赋值了
        }
    }
    return user  // ⚠️ 此时闭包还没执行，返回 nil
}
```

### 问题

- 闭包是**异步**的
- `return user` 在闭包执行**之前**就执行了
- 返回值永远是 `nil`

### 正确代码

```swift
// 方案 1：使用 async/await
func loginAndGetUser() async -> User? {
    let success = await login()
    if success {
        return await getUser()
    }
    return nil
}

// 方案 2：使用回调
func loginAndGetUser(completion: @escaping (User?) -> Void) {
    login { success in
        if success {
            let user = getUser()
            completion(user)
        } else {
            completion(nil)
        }
    }
}
```

### 怎么发现

```bash
你：检查这个函数，是否正确处理了异步
```

## 案例 4：浮点数比较错误

### 场景

检查价格是否为零

### AI 生成的代码

```swift
if price == 0.0 {  // ⚠️ 浮点数不能用 ==
    return "免费"
}
```

### 问题

浮点数运算有精度问题：

```swift
let price = 0.1 + 0.2
print(price == 0.3)  // false！
print(price)  // 0.30000000000000004
```

### 正确代码

```swift
// 方案 1：误差范围
if abs(price) < 0.001 {
    return "免费"
}

// 方案 2：使用 Decimal
let price = Decimal(string: "0.1")! + Decimal(string: "0.2")!
let three = Decimal(string: "0.3")!
print(price == three)  // true
```

## 预防清单

| 检查项 | 方法 |
|--------|------|
| 条件判断 | 用 true/false 用户测试 |
| 循环边界 | 用边界值测试（0、1、最大） |
| 异步代码 | 检查是否用了 async/await |
| 浮点数 | 用小数进行测试 |

---

## 调试技巧

### 1. 添加日志

```swift
func canAccessPremium(user: User) -> Bool {
    print("检查权限: isVIP=\(user.isVIP)")  // 调试日志
    return user.isVIP == false  // 看看日志输出什么
}
```

### 2. 写测试用例

```swift
func testVIPAccess() {
    let vipUser = User(isVIP: true)
    let result = canAccessPremium(user: vipUser)
    assert(result == true)  // 断言
}
```

### 3. 小心 AI 的「修复」

AI 有时会改对地方，但也可能改错：

```
你：这个函数有问题
AI：抱歉，我修复了

修复后的代码...  // ⚠️ 可能引入新问题
```

## 总结

```
⚠️ 能跑 ≠ 正确

✅ 防御式检查
1. 条件判断 → 用正反用例测试
2. 循环边界 → 用 0、1、最大值测试
3. 异步代码 → 确保 await/回调正确
4. 浮点数 → 用小数测试，避免 ==

✅ 调试方法
1. 添加日志
2. 写测试
3. 分步验证
```
