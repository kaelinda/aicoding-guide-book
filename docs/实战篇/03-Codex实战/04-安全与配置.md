# 04-Codex 安全与配置

> 审批模式、敏感操作确认、黑名单命令、常见问题、推荐工作流。

---

## 审批模式

Codex 写文件 / 跑命令前会（默认）请求确认。三档可选：

```bash
# 自动模式（信任的环境，如 CI 或个人玩具项目）
codex --approval auto

# 交互模式（默认，敏感操作需要手动确认）
codex --approval interactive

# 只读模式（只分析、不修改任何文件）
codex --approval readonly
```

### 经验

- **第一次跑新项目用 `interactive`** —— 看它具体想执行什么
- **跑了几天熟悉后用 `auto`** —— 但给项目 `AGENTS.md` 里加足黑名单
- **代码审查 / 文档分析用 `readonly`** —— 防止"顺手修一下"
- **生产环境 / 部署相关不要用 `auto`** —— 永远保持 `interactive`

## 敏感操作确认

`interactive` 模式下：

```text
Codex：我需要执行以下命令：
        npm install some-third-party-package

        来源：未经项目记录的新依赖
        目的：要用它的 JSON schema 验证

        是否继续？

[Approve] [Reject] [View Details]
```

### Approve / Reject 的判断

- **新依赖** → 默认 Reject，问它"为什么必须、能不能不用"
- **`rm` / 批量改文件** → Reject，让它逐个列出再决定
- **网络请求 / 下载外部脚本** → 默认 Reject，除非你完全认可那个域
- **`git push` / `git reset --hard`** → **永远** Reject；让你自己来

## 黑名单命令

在 `AGENTS.md` 里配置（Codex 永远不会执行）：

```markdown
## 安全配置

禁止执行：
- rm -rf
- sudo
- git push --force
- git reset --hard
- npm publish
- pod trunk push
- xcrun altool --upload-app
- fastlane deliver
```

### 推荐的最小黑名单（iOS 项目）

```markdown
## 安全配置（AGENTS.md）

禁止执行：
- rm -rf        # 防止误删
- sudo          # 永远不走 root
- git push --force / git push -f        # 防止毁掉远端历史
- git reset --hard                       # 防止丢本地改动
- pod trunk push                         # 防止意外发布
- fastlane deliver                       # 防止意外提审
- xcrun altool --upload-app              # 防止意外上传到 App Store
- security delete-keychain               # 防止误删钥匙串
```

> 📌 `AGENTS.md` 的黑名单是**字符串前缀匹配**，不是 shell 语法。`rm` 会拦住 `rm file.txt` 但不会拦 `/usr/bin/rm`，所以关键命令可以多写几个变体。具体匹配规则以 Codex 官方文档为准。

## 常见问题

### ❌ 登录失败

```text
错误：Authentication failed
```

```bash
# 1. 检查网络
curl -I https://api.openai.com

# 2. 检查订阅状态
#    访问 https://chat.openai.com/ 看账号是否是 Plus/Pro/Team

# 3. 清 cache 重登
codex --logout
rm -rf ~/.codex/cache
codex
```

---

### ❌ 响应很慢

**常见原因**：推理等级开太高、任务太复杂、网络不稳。

```text
# 降推理等级
/reasoning low

# 换模型
/model gpt-5.3-codex

# 简化任务（拆成多步）
```

---

### ❌ 无法修改文件

```text
原因：权限不足（文件只读 / 不在 workspace 内 / 审批模式是 readonly）
```

```bash
# 检查文件权限
ls -l path/to/file.swift

# 如果是文件只读
chmod 644 path/to/file.swift

# 检查审批模式是不是 readonly
codex --approval interactive
```

---

### ❌ 生成的代码风格不对

**原因**：缺 `AGENTS.md`，Codex 按通用偏好生成。

```bash
# 方案：在项目根建 AGENTS.md（参见 [02-高级功能](02-高级功能.md)）
touch AGENTS.md
```

## 推荐工作流

### 日常开发（一个人）

```text
1. cd 进项目 → codex
2. 描述任务（引用 @文件名 提供上下文）
3. Codex 出方案 → 你 review → 批准执行
4. 每个小阶段跑一次 xcodebuild 验证
5. 一个任务做完 /clear 开新 session
6. git commit（AI 协助写 message）
```

### 和其他 AI 工具配合

**不要都用同一个**。不同工具擅长不同：

| 场景 | 首选 |
|------|------|
| **Tab 补全 / 日常写代码** | Cursor |
| **终端脚本 / 自动化** | Claude Code |
| **复杂推理 / 架构决策** | Codex（`/reasoning high` 时） |
| **代码审查（独立视角）** | 换一个你平时不用的 AI 工具跑一次（交叉验证） |

```text
Cursor（编码打字）
    ↓
Codex（重构 + 审查）
    ↓
Claude Code（运行自动化测试 / 出 commit）
    ↓
真人 review
```

### CI 里跑 Codex

**原则**：

- 只做 review 和报告，**不自动改代码、不挡 merge**
- 结果 post 成 PR comment，作为人类 reviewer 的辅助视角
- 用独立的 `OPENAI_API_KEY`，别共享个人 key
- 设 token 预算上限

详见 [02-高级功能 § CI/CD 集成](02-高级功能.md#4-ci--cd-集成)。
