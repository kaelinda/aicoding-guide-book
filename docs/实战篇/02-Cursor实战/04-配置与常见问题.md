# 04-Cursor 配置与常见问题

> 推荐设置、实用扩展、三个最常见的问题。

---

## 推荐设置

`Cmd + ,` 打开 Settings，在 `settings.json` 里：

```jsonc
{
  // ===== AI =====
  "cursor.aiModel": "claude-3.5-sonnet",   // 或 "gpt-4o"，按订阅选
  "cursor.tabAutocomplete": true,
  "cursor.autocompleteDelay": 100,          // 毫秒。嫌太激进调大

  // ===== 字体 =====
  "editor.fontFamily": "SF Mono, Menlo, Monaco, monospace",
  "editor.fontSize": 14,
  "editor.fontLigatures": true,

  // ===== Swift 专用 =====
  "[swift]": {
    "editor.formatOnSave": true,
    "editor.tabSize": 4,
    "editor.insertSpaces": true
  },

  // ===== 搜索排除（避免 AI 读到一堆 derived data）=====
  "files.exclude": {
    "**/.git": true,
    "**/DerivedData": true,
    "**/Pods": true,
    "**/.build": true,
    "**/*.xcodeproj": false   // 保留，让 AI 能看到工程配置
  },
  "search.exclude": {
    "**/DerivedData": true,
    "**/Pods": true,
    "**/.build": true
  }
}
```

### 小调整（按需）

```jsonc
{
  // 开启 vim 模式（如果你来自 Vim）
  "vim.enable": true,

  // 自动保存（改动跟着 AI 生成自动落盘，适合"让它随便改"的阶段）
  "files.autoSave": "afterDelay",
  "files.autoSaveDelay": 1000,

  // 关闭代码补全的小气泡提示（太吵）
  "editor.inlineSuggest.showToolbar": "onHover"
}
```

## 推荐扩展

| 扩展 | 用途 |
|------|------|
| **Swift (Apple)** | 官方 Swift 语言服务 |
| **SwiftFormat for Xcode / sweetpad** | 代码格式化 |
| **SwiftLint** | 代码规范检查（需项目内 `.swiftlint.yml`） |
| **GitLens** | Git 增强（作者、blame、history） |
| **Error Lens** | 错误直接显示在当前行末尾 |
| **Code Spell Checker** | 拼写检查（常抓 Swift 变量名 typo） |
| **Better Comments** | 高亮 `// TODO` / `// FIXME` / `// NOTE` |

### iOS 专用建议

- **sweetpad** 扩展能在 Cursor 里直接 build / run 模拟器（需配合 `xcodebuild`）
- 如果你用 Tuist / XcodeGen，装对应的**语法高亮扩展**，编辑 `Project.swift` / `project.yml` 更舒服

## 常见问题

### ❌ AI 响应很慢

**可能原因**：网络、服务器负载、上下文太长。

```
1. 检查网络，必要时开代理：
   export https_proxy=http://127.0.0.1:7890

2. 换模型（Settings → Cursor → AI Model）：
   - 不够快：Claude 3.5 Sonnet → Haiku
   - 需要聪明：升到 Opus / GPT-4o

3. 简化提示词，`Cmd + L` 开新对话而不是堆叠长上下文
```

### ❌ 代码补全不准确

**原因**：缺项目上下文。

```
1. 项目根建 .cursorrules 文件，写明技术栈（见 [03-高级技巧](03-高级技巧.md)）
2. 提示词里明确 iOS 版本、SwiftUI / UIKit 偏好
3. 用 @符号 引用相关文件，给 AI 上下文（@UserService）
4. 检查 .cursorignore，别把关键源码排除了
```

### ❌ 无法打开 / 运行 iOS 项目

Cursor 是 VSCode 衍生，**不是 Xcode**。它不能替代 Xcode 做：

- 编译 / 运行到模拟器和真机
- 签名、证书、profile 管理
- Instruments 性能分析
- Storyboard / xib 可视化编辑

正确工作流：

```
1. 用 Cursor / Claude Code 写代码
2. 用 Xcode 编译、运行、调试、签名
3. 两边同时开着，代码改动会自动互相看到
```

如果想在终端 build：

```bash
# 编译（无 GUI）
xcodebuild build -scheme MyApp -destination 'platform=iOS Simulator,name=iPhone 15'

# 测试
xcodebuild test -scheme MyApp -destination 'platform=iOS Simulator,name=iPhone 15'

# 清 DerivedData（解决玄学问题常用）
rm -rf ~/Library/Developer/Xcode/DerivedData
```

## 和 Claude Code 配合使用

Cursor 和 Claude Code **不是二选一**。推荐组合：

| 场景 | 用哪个 |
|------|------|
| 写代码、日常编辑、补全 | Cursor |
| 复杂重构、跨文件批量改 | Claude Code（终端）或 Cursor Composer |
| 自动化 / CI / 脚本 | Claude Code（无 GUI 依赖） |
| 需要浏览项目结构 | Cursor（侧栏方便） |
| 长对话 / 多轮决策 | Claude Code（上下文管理好） |

### 共用配置

- `.cursorrules` 和 `CLAUDE.md` **可以都有**
- 重复内容用 symlink 或简单复制都行
- 个人偏好：CLAUDE.md 放技术栈和铁律，.cursorrules 放 IDE 行为相关的偏好
