# 13-Claude Code 我的常用 Prompt

> 我日常最常用的一组高频 Prompt。

---

## 1. 项目理解

```text
先不要改代码，分析项目结构、技术栈、核心模块
```

---

## 2. Bug 排查

```text
先判断问题类型：状态/并发/生命周期/网络
```

---

## 3. 最小修改

```text
只做最小必要修改，不改无关代码
```

---

## 4. 方案设计

```text
先给实现方案，再执行
```

---

## 5. 拆任务

```text
把任务拆成 3-5 步再开始
```

---

## 6. 验证

```text
改完后运行测试或验证步骤
```

---

## 7. 总结

```text
总结改动、风险、后续建议
```

---

一句话总结：
把常用操作沉淀为固定 Prompt
