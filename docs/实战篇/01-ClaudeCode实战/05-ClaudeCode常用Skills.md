# 05-Claude Code 常用 Skills 清单

> 一组可以直接复用的高质量 Claude Code 使用场景（类似技能插件）。

---

## 什么是 Skills

在 Claude Code 里，Skills 可以理解为：

> 一段结构化的任务指令 + 约束 + 输出要求

用来反复解决同一类问题。

---

## 1. 项目理解 Skill

```text
请先不要修改任何代码。
完成以下分析：
1. 项目目录结构
2. 技术栈
3. 核心模块划分
4. 主要入口
5. 可能的架构模式
最后输出一份简洁总结。
```

适用：新项目、陌生仓库

---

## 2. Bug 排查 Skill

```text
请先不要改代码。
帮我分析这个问题：
- 现象：...
- 复现步骤：...
- 相关代码：...

请判断问题更可能属于：
- 状态
- 并发
- 生命周期
- 数据边界
- 网络
并说明理由。
```

---

## 3. 最小修复 Skill

```text
请修复这个问题，但遵守：
- 只做最小必要修改
- 不改无关文件
- 不调整结构
- 不引入新依赖
修改后运行相关测试或构建。
```

---

## 4. MVVM 重构 Skill

```text
请将这个模块重构为 MVVM：
- View 只负责展示
- ViewModel 负责状态
- 网络逻辑移到 Repository
- 保持功能不变
先给出重构计划，再执行。
```

---

## 5. 批量修改 Skill

```text
请扫描项目中所有 [某类文件]：
- 找出符合条件的代码
- 统一修改为新写法
- 输出修改文件列表
- 总结风险
```

适用：
- API 替换
- 主题替换
- 命名规范统一

---

## 6. 测试补全 Skill

```text
请为这个模块补充测试：
- 正常流程
- 边界情况
- 错误路径
使用 Mock 或依赖注入。
```

---

## 7. 性能优化 Skill

```text
请从性能角度分析这段代码：
- 是否存在主线程阻塞
- 是否有重复计算
- 是否有不必要渲染
- 是否有内存问题
给出优化建议。
```

---

## 8. 代码审查 Skill

```text
请审查这个模块：
- 是否有坏味道
- 是否有潜在 bug
- 是否有可维护性问题
- 是否有测试缺失
输出改进建议。
```

---

## 9. 发布前检查 Skill

```text
请检查这个项目是否可以发布：
- 是否有调试代码
- 是否有敏感信息
- 是否有明显性能问题
- 是否有未处理错误
输出检查清单。
```

---

## 10. 项目总结 Skill

```text
请总结本次改动：
- 修改了哪些文件
- 每个修改的目的
- 是否有风险
- 后续建议
```

---

## 推荐用法

你可以把这些 Skills：
- 存在本地
- 写进 AGENTS.md
- 或作为日常固定模板

---

## 小结

高效使用 Claude Code 的关键，不是“每次重新想 prompt”，而是：

> 把常见任务沉淀成可复用的 Skills

这样你会越来越快。
