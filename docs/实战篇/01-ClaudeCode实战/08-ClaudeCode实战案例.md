# 08-Claude Code 实战案例

> 把前面的技巧、上下文、记忆、skills 串起来，用真实开发流程跑一遍。

---

## 场景：修复 iOS 登录页重复请求

### Step 1：定义任务

```text
当前任务：修复登录页重复请求

任务边界：
- 只处理登录页
- 不修改网络层
- 不引入新依赖
```

---

### Step 2：让 Claude 分析

```text
先不要改代码：
- 阅读 LoginView 和 ViewModel
- 判断问题类型
- 给出可能根因
```

Claude 输出：
- 可能是 SwiftUI 状态问题
- 或生命周期触发两次请求

---

### Step 3：给修复方案

```text
给出最小修复方案，并说明改动范围
```

Claude：
- 调整 StateObject
- 收敛请求触发入口

---

### Step 4：执行修改

```text
只做最小修改，改完后运行验证
```

---

### Step 5：总结

```text
总结这次改动：
- 改了哪些文件
- 根因是什么
- 如何避免再次发生
```

---

## 场景：新增一个页面（SwiftUI）

### Step 1：先看项目

```text
先分析项目结构和已有页面模式
```

### Step 2：给计划

```text
拆分步骤：
1. 创建 View
2. 创建 ViewModel
3. 接入 API
4. 添加状态处理
```

### Step 3：实现最小版本

```text
先实现最简单版本，确保能跑
```

### Step 4：补边界

- 加载态
- 错误态
- 空态

---

## 场景：代码审查

```text
请审查这个模块：
- 是否有状态问题
- 是否有重复请求
- 是否有主线程阻塞
```

Claude 输出：
- 风险点
- 优化建议

## 场景：性能优化

```text
请分析这个页面卡顿原因
```

- 指出 body 中计算

## 一句话总结流程

Claude Code 实战的核心不是 prompt，而是流程：

1. 定义任务
2. 分析问题
3. 给方案
4. 小步执行
5. 验证结果
6. 总结沉淀
