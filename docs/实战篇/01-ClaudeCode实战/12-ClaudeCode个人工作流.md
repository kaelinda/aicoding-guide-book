# 12-Claude Code 个人工作流

> 我日常如何使用 Claude Code 的标准流程。

---

## 一句话工作流

先分析 → 再计划 → 小步执行 → 每步验证 → 最后总结

---

## 修 bug 流程

1. 定义问题
2. 判断问题类型（状态/并发/生命周期/网络）
3. 最小修复
4. 验证结果

---

## 新功能流程

1. 分析项目结构
2. 设计方案
3. 拆任务
4. 实现最小版本
5. 补边界
6. 优化

---

## 重构流程

1. 给两个方案
2. 选择保守方案
3. 分轮执行
4. 每轮验证

---

## 我的原则

- 一轮只做一件事
- 控制改动范围
- 必须验证
- 每轮总结

---

一句话总结：
用 AI 执行工程流程，而不是写代码
