# 06-从零生成功能模板

> 当你想让 AI 从 0 开始帮你做一个功能时，如何避免“一上来就生成一大坨”

## 🎯 这个模板适合什么时候用

适用于这些场景：

- 你准备做一个新功能
- 你现在还没有现成实现
- 你希望 AI 从需求出发逐步帮你搭起来
- 你不想让 AI 一次性生成一个你自己都看不懂的大工程

## 🤔 为什么“从零生成”特别容易失控

因为这类任务最容易出现下面这些问题：

- 需求还没讲清楚就开始写代码
- AI 默认做了很多你没要求的设计
- 一次生成过多文件和结构
- 你很难判断第一版到底哪里该改

所以，从零生成最重要的不是“让 AI 直接开始写”，而是让它先收敛范围、明确阶段、控制复杂度。

## 🧩 通用模板

```text
我想从 0 开始实现一个新功能，请按分阶段方式协助我。

项目背景：
- 技术栈：{例如 SwiftUI + SwiftData}
- 当前项目类型：{例如 iOS Todo App}
- 我现在要做的功能：{例如 设置页 / 搜索页 / 登录页}

目标：
{用 1~3 句话描述你想完成什么}

请按下面顺序协助我：
1. 先帮我定义最小可行版本
2. 再拆成 3~5 个实现步骤
3. 先实现第一步，不要一次生成全部内容
4. 每一步都说明验证方式

要求：
- 优先简单、清晰、适合初学者理解
- 不要过度设计
- 不要一次生成完整大项目
- 如果有多种实现方案，请先给推荐方案并说明原因
```

## ✅ 推荐输出顺序

理想情况下，AI 最好按这个顺序输出：

1. 最小可行版本定义
2. 实现步骤列表
3. 第一步代码
4. 第一步验证方式
5. 下一步建议

这会比“直接输出所有文件”更稳定得多。

## 💬 适合继续追问的方向

### 追问 1：缩小功能范围

```text
请先收敛到最小可行版本，不考虑高级功能。
```

### 追问 2：只做第一步

```text
现在先只实现第一步，不要继续展开后面的部分。
```

### 追问 3：要求解释为什么这样拆

```text
请解释为什么你建议按这个顺序实现。
```

### 追问 4：要求补验收清单

```text
请为第一步补一份最小验收清单。
```

## 📱 iOS 示例

### 示例：从 0 生成一个设置页

```text
我想从 0 开始实现一个新功能，请按分阶段方式协助我。

项目背景：
- 技术栈：SwiftUI
- 当前项目类型：iOS App
- 我现在要做的功能：设置页

目标：
创建一个简单的设置页，包含深色模式开关、通知开关和关于页面入口。

请按下面顺序协助我：
1. 先帮我定义最小可行版本
2. 再拆成 3~5 个实现步骤
3. 先实现第一步，不要一次生成全部内容
4. 每一步都说明验证方式

要求：
- 优先简单、清晰、适合初学者理解
- 不要过度设计
- 不要一次生成完整大项目
```

## ⚠️ 使用这个模板时的注意事项

### 注意 1：不要把“以后也许会做的功能”全塞进去
先做最小版本，后续再加。

### 注意 2：不要一上来就要求完整架构
架构要和复杂度匹配，不是越复杂越好。

### 注意 3：一定要控制输出节奏
从零生成时，“分阶段”比“全量生成”重要得多。

## ✅ 小结

从零生成功能时，最值得掌握的不是某个神奇 Prompt，而是这套节奏：

- 先收敛需求
- 再拆步骤
- 再逐步实现
- 每步都验证

记住这句话：

> **从 0 开始时，控制复杂度比追求一步到位更重要。**
