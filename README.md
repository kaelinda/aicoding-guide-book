# AI Coding Guide Book 📚

> 面向初级 iOS 开发者的 AI 编程实践指南  
> 从环境搭建到多 Agent 协作，系统掌握 AI 辅助开发的核心方法

---

## 🎯 这是什么

`AI Coding Guide Book` 是一份围绕 **AI 辅助编程** 的系统化教程，重点面向：

- 有基础 Swift / Xcode 使用经验的 iOS 开发者
- 想把 AI 真正用进日常开发流程的独立开发者
- 对 Claude Code、Cursor、Codex、MCP、Agent 协作感兴趣的学习者

这不是一份单纯的工具说明书，而是一套从 **入门体验、核心概念、实战方法，到高阶工作流** 的完整学习路径。

---

## 👥 适合谁读

这份指南尤其适合下面几类读者：

- 想开始尝试 AI 编程，但不知道从哪里入手
- 会写 iOS App，但还不会高效和 AI 协作
- 已经在用 Cursor / Claude Code / Codex，但流程不稳定
- 希望建立更清晰的 Prompt、验收、修复和自动化工作流
- 想把 AI 工具从“偶尔试试”变成“每天都能用”

---

## 🚀 你将学到什么

通过这份指南，你将逐步掌握：

- 如何搭建完整的 AI 编程环境
- 如何选择适合自己的 AI 编程工具
- 如何把需求清楚地交给 AI
- 如何评估、修正和验收 AI 生成的代码
- 如何把 AI 工具用到真实 iOS 开发场景中
- 如何进一步构建多 Agent 协作和自动化工作流

## ✨ 这份指南的特点

### 1. 面向真实开发，而不是空谈概念
本项目强调“能落地”。每一部分都尽量围绕真实开发问题展开，而不是只介绍工具功能。

### 2. 面向初级 iOS 开发者
你不需要先成为 AI 专家。只要你已经会一些 Swift / Xcode，就可以逐步把 AI 引入自己的开发流程。

### 3. 强调“生成之后的判断力”
AI 能生成很多代码，但真正重要的是：

- 你是否能判断它对不对
- 你是否知道哪里要修改
- 你是否能建立自己的验收标准

### 4. 关注长期可持续的工作流
除了“怎么生成代码”，这份指南也会覆盖：

- Prompt 组织方式
- 代码验收方法
- 安全边界设置
- 多工具协作
- 自动化工作流

## 📝 阅读建议

为了获得更好的学习效果，建议你采用下面的方式：

### 不要只读，要跟着做
每学完一章，至少自己动手完成一个小练习。

### 不要只让 AI 生成，要学会验收
重点不是“让 AI 写得更快”，而是“写完后你知道怎么判断”。

### 不要一口吃完
你不需要一次性读完整本。更好的方式是：

- 先上手
- 再解决当前最需要的问题
- 最后回头补概念和体系
